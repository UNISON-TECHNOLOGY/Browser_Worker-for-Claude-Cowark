運用ダッシュボードを生成・更新してください。

## データソース（あるものだけ使う。無いセクションは「未運用」として1行表示）

| セクション | ソース | 表示 |
|---|---|---|
| 進行中タスク | memory/.workflow/ + memory/session-log.md 末尾 | タスク名・フェーズ・引き継ぎ事項 |
| 媒体サマリー | knowledge/media/registry.yaml + status/ 最新 | 媒体別の主要数値・前回比・アラート（残数/期限） |
| SNSストック | knowledge/sns/*/queue.md + strategy.md | プラットフォーム別ストック充足率（目標に対する%） |
| 競合ウォッチ | knowledge/watch/ 最新差分 | 直近の変更検知（重要度順・上位5件） |
| サイト健康 | knowledge/audits/ 最新 | 自社サイトの LCP/CLS 等の最新値と前回比 |
| 定常タスク実績 | knowledge/logs/ 直近 | 直近7日の実行履歴（成功/失敗/スキップ） |
| 学習記録 | knowledge/feedback/lessons.md | 直近に追加された OK/NG（3件まで） |
| **OSの成長** | .claude/skills/ + knowledge/tacit/ + knowledge/sites/ + lessons.md | 育ちの可視化（下記） |
| **できることガイド** | プラグインの commands/ 一覧 | インフォメーションガイド（下記） |
| **生成物ライブラリ** | knowledge/artifacts-index.md | 発行済みアーティファクトへのリンク集（下記） |

## 生成ルール

1. **templates/dashboard-template.html（「秘湯紀行」テーマ）を骨格に使う**（deliverable-writer に委譲）。**タブはタスク単位**: 「全体」+ ワークスペースの定常タスクごとに1タブ（タスクが雛形と異なる場合はタブを増減し、場面はテンプレの6場面を順に割り当てる）。各タスクタブに状態カード（フェーズ①〜④/現在ステップ/次回実行/状態）と工程の飛び石図（A〜K・関所チップ）を必ず出し、見本データだけを実データに差し替える。CSS・構造・演出は変更しない。**ラベルは平易な機能名のみ・世界観は視覚だけで表現・ファーストビュー（全体タブ）でアラート+タスク一覧に到達**（大原則は templates/dashboard-design.md）。report-template.html は単発のデータレポート用でありダッシュボードには使わない
1b. 推移・履歴データは knowledge/data/delvework.db から取得し、**JSONとしてページ内に埋め込んで**JSで絞り込み・ソート・期間切替できるようにする（アーティファクトは外部通信不可のため、データは焼き込みが正。数千行までは全量埋め込んでよい）
2. **アラートを最上部に集約**: 媒体残数警告・契約更新30日前・SNSストック目標割れ・競合の重要変更・未完了タスク。ゼロなら「アラートなし ✅」
3. 各セクションに「次の一手」を1行添える（例: 「ストック残1日分 → 『Xの投稿ストック埋めて』で充足」）— ダッシュボードから次の指示文をコピペできる導線
4. 生成日時を明記。ローカルにも `knowledge/reports/dashboard.html` として保存（毎回上書き）

## 新セクションの生成ルール

### OSの成長（「自分専用OSに育っている」の可視化）

- カウントを stat タイルで: 習得スキル数（ワークスペース .claude/skills/）/ 暗黙知トピック数（knowledge/tacit/）/ 攻略済みサイト数とフェーズ内訳（knowledge/sites/ — ①初回〜④最適化）/ 学習済み OK/NG 数（lessons.md）
- 増分を delta 表示（前回ダッシュボード生成時との差。「今週 +2 スキル」）
- 直近に習得したスキル・暗黙知を3件、名前+「言い方」つきでリスト（「使ってみる」導線）

### できることガイド（インフォメーション）

- プラグインの commands/*.md の frontmatter（description）から「やりたいこと → 言い方の例 → コマンド名」の一覧表を生成
- ワークスペースで自動生成されたスキル（.claude/skills/）も同じ表に「習得スキル」バッジつきで併記
- カテゴリ分け: 調査系 / 制作系 / 運用系 / 学習系 / その他

### 生成物ライブラリ

- `knowledge/artifacts-index.md`（発行台帳）を読み、日付降順のリンク集として表示（種別バッジ: レポート/モックアップ/ガイド）
- ローカルにしかない成果物（未発行）は「ローカルのみ」と注記してパス表示

## アーティファクト発行（重要）

- create_artifact / update_artifact が使える環境では、**初回は create、以降は必ず同じアーティファクトを update** する（URL を固定し、ブックマーク可能な「司令塔ページ」にする）
- アーティファクトの ID/URL は `knowledge/media/dashboard-artifact.md` に記録し、次回の update で参照する
- 発行後、共有URLを報告する

## 運用の推奨

- 定常タスク（watch / media status / sns）の**締めに毎回このコマンドを実行**すると、ダッシュボードが常に最新に保たれる（スケジュール登録時はルーチンの最後に入れる）
- 設定の変更（媒体登録・ストック目標・スケジュール）はダッシュボードからはできない。変更はチャットで指示する（ダッシュボードの「次の一手」がその入口）
