この環境の操作ガイドを生成してください。対象読者: $ARGUMENTS

## 内容（この順で）

1. **これは何か**（3行）: ブラウザ定常業務を安全に自動化し、使うほど育つ仕組み。コマンドを覚える必要はないこと
2. **やりたいこと別ガイド（本体）**: プラグイン commands/*.md の description から「やりたいこと → そのまま言う一言 → 何が起きるか」の表をカテゴリ別に生成
   - カテゴリ: 調査する / 作る / 運用する（定常業務）/ 学ばせる / 設定・確認
   - **packs.conf で無効のパックはグレー表示+「OFF」バッジ**（存在は見せるが使えないことを明示）
   - ワークスペースの習得スキル（.claude/skills/）も「この環境で覚えた技」として同じ形式で載せる
3. **安全の仕組み**（図解1つ）: ゲート（勝手に送信しない）→ 承認 → 実行 → 記録、の流れ。Credential Guard（パスワードは人間だけ）も1行
4. **育ち方**: 使うほど何が溜まるか（地図/手順/暗黙知/好み）と、育ちを見る場所（ダッシュボード）
5. **困ったとき**: ブロックされた→ /delve-status、状況確認→ダッシュボード、機能ON/OFF→「◯◯パックをONにして」

## 生成ルール

- **templates/guide-template.html（和風総本家テーマ）を骨格に使う**。設計意図と禁じ手は templates/guide-design.md（正本）に従う。report-template.html はデータレポート用でありガイドには使わない
- deliverable-writer 委譲。読者は非エンジニア前提、専門用語は言い換える（「エージェント」→「職人」等の言い換えは guide-design.md の世界観に合わせてよいが、コマンド名・設定名は正確に）
- 保存: knowledge/reports/user-guide.html（上書き）
- **アーティファクト発行は固定URL**: 初回 create、以降は同じアーティファクトを update（URL は knowledge/media/guide-artifact.md に記録）。artifacts-index.md に追記
- コマンド・パック・スキルが変わるたびに再生成すれば常に実態と同期したガイドになる（ダッシュボード更新時に同時更新を推奨）
