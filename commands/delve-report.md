Delvework タスク「$ARGUMENTS」（未指定なら `memory/.workflow/active` の直近タスク）のブラウジングレポートを HTML で生成してください。

## 手順

1. 材料を集める:
   - `memory/.workflow/`（タスク名・フェーズ）
   - `knowledge/logs/` の該当タスクのログ
   - `memory/session-log.md` の該当セクション
   - セッション中に取得したスクリーンショット（あれば）
2. `knowledge/reports/<task-name>-<date>.html` として自己完結型 HTML を書き出す（CSS/JS インライン、外部リソース参照なし）

## レポートに含める内容

| セクション | 内容 |
|-----------|------|
| ヘッダー | タスク名・実行日・フェーズ（①初回/②再訪問/③構造変更/④最適化）・対象サイト |
| サマリー | 何をして何が変わったか3行以内。成功/失敗/部分成功のバッジ表示 |
| 操作タイムライン | 実行した操作の時系列リスト（navigate→click→fill...）。各操作に対象要素と結果 |
| 変更前後の比較 | Step E（変更前記録）と Step J（変更後）の差分。スクショがあれば並置 |
| 地図の成長 | 今回のタスクで knowledge/sites/ に追加・更新されたナレッジの一覧 |
| 引き継ぎ | 未完了事項・次の探索者への知見（session-log から抽出） |

## デザイン指針

- ダーク/ライト両対応（`prefers-color-scheme`）
- 印刷・共有を想定したシンプルな1カラム。横スクロールは表・コード内のみ
- フェーズごとにアクセントカラーを変える（①青=探索 ②緑=再訪 ③橙=変更検知 ④紫=最適化）

生成後、ファイルパスを報告し、可能な環境ならブラウザで開いて見せること。

## モデル最適化

材料集め（ログ・フラグ・スクショの収集）まではメインループで行い、**HTML の執筆は `deliverable-writer` エージェント（Agent ツール）に委譲する**こと。委譲時は材料ファイルの絶対パスと出力先パスをプロンプトに明記する。
