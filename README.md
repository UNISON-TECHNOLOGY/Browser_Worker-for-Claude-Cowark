# Browser Worker for Claude Cowork

> *Delve into the unknown. Map by touch. Master through repetition.*

**Delvework（デルヴワーク）** — AI がブラウザ業務を「探索 → マッピング → 最適化 → 実行」の
方法論で安全に自動化する Claude Cowork プラグイン。使うほどにナレッジ（地図）が溜まり、
同じ業務が速く・正確になっていきます。

## セットアップ（2ステップ）

1. Cowork の **設定 → プラグイン** でこのリポジトリを marketplace として追加:
   `https://github.com/UNISON-TECHNOLOGY/Browser_Worker-for-Claude-Cowark`
2. **browser-worker** を有効化（「ローカルMCPサーバーを含む」ダイアログは「続ける」）

ブラウザ操作は **Claude in Chrome**（あなたの実ブラウザ）で行うため、ログイン済みの
サイトがそのまま使えます。チャットの「+」→ コネクタで Claude in Chrome を ON にして
ください。

## 使い方 — コマンドを覚える必要はありません

普通に日本語で頼めば、Claude が適切な機能を自動で使います。

| やりたいこと | 言い方の例 |
|---|---|
| サイトの状態を調べる | 「このサイトの表示速度を測って」「リンク切れないかチェックして」 |
| デザインの下調べ | 「競合3社のLPのデザインを分析して」「このサイトの配色とフォントを調べて」 |
| ブラウザ業務の実行 | 「○○サイトのイベント情報を更新して」「フォームに入力して」 |
| 作業の報告 | 「今日の作業をレポートにまとめて」 |
| 状態確認 | 「前回の続きはどこから？」「ナレッジ何が溜まってる？」 |
| ページ改善・モックアップ | 「このLPを改善して」「リニューアル案を作って」 |
| 文章作成 | 「求人票を書いて」「キャッチコピー10案」「提案書のドラフト」※人材業界向けパック |

明示的に呼びたい場合のコマンド: `/delve-start` `/delve-status` `/delve-style`
`/delve-audit` `/delve-report`

## 何が起きるか（Delvework の動き）

1. **変更操作は勝手に実行されません** — サイトへの入力・クリック等の変更操作は、
   タスク開始手続きと「変更前状態の記録」を済ませるまでゲートがブロックします
2. **地図が残ります** — 訪れたサイトの構造・手順は `knowledge/` に蓄積され、
   次回同じ業務が速くなります（初回→再訪問→最適化とフェーズが上がる）
3. **成果はHTMLレポート** — 調査・診断・作業報告は統一デザインのレポートで出力
   （デジタル庁ダッシュボードガイドブック準拠のテンプレート）
4. **高品質な執筆は Opus に委譲** — 通常操作は軽く、成果物の執筆だけ上位モデルが担当

## 継続利用する場合（推奨）

「フォルダを追加」で業務用フォルダを接続してください。ナレッジとログがフォルダに
永続化され、セッションを跨いで引き継がれます。未接続でも動きますが、蓄積が
セッション終了で消えます。

## 同梱物

| 種類 | 内容 |
|---|---|
| コマンド 7 | delve-start / status / style / audit / report / improve（自然文から自動発火） |
| 日本語コマンド 6 | /サイト診断 /スタイル調査 /ページ改善 /タスク開始 /状態確認 /作業レポート |
| フック 3 | 変更操作ゲート / ナビゲーション警告 / セッション開始時の引き継ぎ通知 |
| 汎用スキル 1 | web-design（LP/UI設計・CRO・モーション。リソース6本） |
| 業種スキル 6 | recruit / copy / sales / logical / business / storytelling（**人材業界パック** — 業種に合わせて差し替え可能な例） |
| エージェント 3 | deliverable-writer（レポート執筆・sonnet）/ design-artisan（モックアップ生成・fable）/ design-critic（デザイン審査・sonnet） |
| テンプレート | HTMLレポート骨格 + デザイン原則（デジタル庁ガイドブック準拠） |
| MCP | Playwright（ローカル Claude Code で使う場合のフォールバック） |

## 業種カスタマイズ

ライティング6スキルは人材業界向けの実装例です。他業種で使う場合は `skills/` 配下の
該当スキルを自業種のトーン・テンプレート・NG表現に書き換えてください（frontmatter の
`description` の「Use when」を業務の言い方に合わせるのがポイント）。web-design /
コマンド / フック / テンプレートは業種非依存でそのまま使えます。

## 安全設計

- 認証情報は扱いません。ログインは人間が行います
- 変更操作はワークフローゲート（フェーズ判定・変更前記録）を通過しないと実行されません
- 検証状況・既知の制約は [TESTING.md](TESTING.md) を参照
