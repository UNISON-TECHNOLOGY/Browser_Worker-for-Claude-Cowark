# Browser Worker for Claude Cowork

> *Delve into the unknown. Map by touch. Master through repetition.*

ブラウザワーカーは **Delvework をして、Forgecraft を返します**。

- **Delvework（デルヴワーク）** = 掘る仕事 — ブラウザを探索し、マッピングし、ナレッジ（地図）を残す。使うほど同じ業務が速く・正確になる
- **Forgecraft（フォージクラフト）** = 鍛える仕事 — 掘り出した素材（データ・画像・知見）を、実証基準と職人エージェントで成果物（レポート・バナー・LP・文章）に鍛え上げて返す

安全設計（変更操作のゲート・送信前監査・人間承認）の上でこの2つが回ります。

## セットアップ（3ステップ）

1. Cowork の **設定 → プラグイン** でこのリポジトリを marketplace として追加:
   `https://github.com/UNISON-TECHNOLOGY/Browser_Worker-for-Claude-Cowark`
2. **browser-worker** を有効化
3. 最初に `/セットアップ` — 運用するSNS・生成AIアカウント・素材サイトをチェックリストで回答（1回だけ。あとから変更可）

ブラウザ操作は **Claude in Chrome**（あなたの実ブラウザ）で行うため、ログイン済みの
サイトがそのまま使えます。チャットの「+」→ コネクタで Claude in Chrome を ON にして
ください。

## 使い方 — コマンドを覚える必要はありません

普通に日本語で頼めば、Claude が適切な機能を自動で使います。

コマンドは**業務ジャンルごとのまとまり（本書では「パック」と呼びます）**で、やってほしいことは
引数に自由に書けます。パック内では要望を「リサーチ / 収集 / クリエイティブ（作る）/ 分析 /
掃き出し（＝投稿・入稿・書き出しなどの仕上げ）」の仕事に振り分け、単発・一連の流れ・
まるごと運用の3つの粒度で実行します。

| カテゴリー（パック） | 言い方の例 |
|---|---|
| /Webサイト | 「表示速度を測って」「デザイン調べて」「このLPを改善して」「徹底的に洗い出して」 |
| /広告 | 「競合広告を洗い出して」「バナー用の素材探して」「Geminiで画像/動画作って」「バナーにして」「WebMにして」「このバナーからLP」「TikTok広告の台本」（セットアップで広告媒体を選ぶと /Google広告 等の専用コマンドが生え、単一媒体はそちらが第一入口） |
| /SNS運用 | 「Xの投稿ストック埋めて」「noteを書いて」（X/Instagram/Threads/TikTok/note/YouTube/LINE対応。セットアップで媒体を選ぶと /X運用 等の専用コマンドが生え、単一媒体はそちらが第一入口） |

はじめての方は「何ができるの？」と聞けばガイドツアー（デモ）が始まります。

## 何が起きるか（Delvework の動き）

1. **変更操作は勝手に実行されません** — サイトへの入力・クリック等の変更操作は、
   タスク開始手続きと「変更前状態の記録」を済ませるまでゲートがブロックします
2. **地図が残ります** — 訪れたサイトの構造・手順は `knowledge/` に蓄積され、
   次回同じ業務が速くなります（初回→再訪問→最適化とフェーズが上がる）
3. **成果はHTMLレポート** — 調査・診断・作業報告は統一デザインのレポートで出力
   （デジタル庁ダッシュボードガイドブック準拠のテンプレート）
4. **成果物ごとに専門AIが分業** — レポート執筆・モックアップ生成・デザイン審査を専任エージェントが担当し、生成→審査→修正のループで品質を確保

## 継続利用する場合（推奨）

「フォルダを追加」で業務用フォルダを接続してください。ナレッジとログがフォルダに
永続化され、セッションを跨いで引き継がれます。未接続でも動きますが、蓄積が
セッション終了で消えます。

## メディア制作を使う場合（必須セットアップ）

画像・動画の生成物をフル解像度で取り込んで加工（背景除去・WebM化・バナー合成）するには
**専用ダウンロードフォルダ**の接続が必要です（2026-07-23 実機検証: Downloads→手動アップロード
経由はレンダリングに失敗する）:

1. PC に専用フォルダを作る（例: `CoworkDrop`）
2. Chrome の設定 → ダウンロード → 保存先をそのフォルダに変更（または「保存場所を確認」を ON にして都度そこへ）
3. Cowork の「フォルダを追加」で **そのフォルダだけ** を接続する

**Downloads フォルダ本体は絶対に接続しない**でください（個人書類が丸ごと AI から見えてしまいます）。

## 自社カスタマイズ

執筆リファレンスは業界を問わず使えます。自社のトーン・NG表現・定型フォーマットに合わせたい場合は
`references/` 配下の該当ファイルを編集してください（例文の差し替えだけでも効果があります）。
コマンド / フック / テンプレートはそのまま使えます。

### 誤ブロックの開放・抑制リスト（あなたが手で編集するファイル）

ガードが安全のために止めすぎたときは、業務フォルダの `knowledge/config/` に
テキストファイルを置いて調整できます。**書式は全ファイル共通**です:

- 1行に1パターン（URLや文言の一部でOK。正規表現も使えます）
- `#` で始まる行はコメント — なぜ開放したかをメモしておくと後で助かります

| ファイル | 効果 |
|---|---|
| `url-allowlist.txt` | URL Guard の誤ブロックを開放（例: 金銭と無関係な `/subscribe` ページ）。**AIは代筆しません** — あなた本人が編集します |
| `money-suppress.txt` | Money Watch が平常画面（例: 常時表示の料金メニュー）に反応しすぎるときの抑制 |
| `critic-suppress.txt` | デザイン審査ゲートを通さなくてよいファイル（例: 単なるスクリーンショット）のパターン |

## 安全設計

- 認証情報は扱いません。ログインは人間が行います。**Credential Guard**: パスワード・認証コード欄へのAI入力は hook が常時ブロック（ワークフロー承認済みでも）
- 変更操作はワークフローゲート（フェーズ判定・変更前記録）を通過しないと実行されません
- 無人運用（PC起動→自動定常タスク）の構成は [docs/unattended-ops.md](docs/unattended-ops.md) を参照（パスワード無保管のまま自動化する3層設計）
- 送信・投稿など不可逆な送出の前は pre-send-verifier（敵対的監査）+ 人間承認の二段ゲート。金銭・契約系画面は **Money Watch** が検知して変更操作を強制停止します
- **既知の限界（設計上の割り切り）**:
  - ゲートは MCP ブラウザツールが対象。**Bash 経由の直接送信（node/curl 等）はゲート対象外** — Node スクリプト等での送信運用はワークスペース側の hook か権限設定で別途ゲートすること
  - 同一ワークスペースの**並行セッションはフラグ（memory/.workflow/）を共有**するため相互干渉しうる。同時実行は1タスクずつを推奨
  - Credential Guard / JS mutation 判定はキーワード検知の best-effort（難読化で回避可能）。硬い防御は Money Watch・URL Guard・人間承認が担う
  - URL denylist は部分一致のため、金銭に無関係な /subscribe 等を誤ブロックすることがある → `knowledge/config/url-allowlist.txt` で開放
  - ゲートの実体は `memory/.workflow/` のフラグファイルで、**Bash からフラグを直接 touch/rm すれば手順を飛ばして技術的に迂回できる**（Log Gate も同様に自己規律で、hook の強制はない）。ゲートは「うっかり・手順飛ばし」への防御であり、意図的な迂回への防御ではない — 意図的迂回が脅威になる環境では、ワークスペース側の権限設定で Bash による `memory/.workflow/` への書き込みを制限すること
  - Money Watch はテキスト読取（read_page 等）の内容に反応する。**スクリーンショットだけで進むと検知面を素通りする**ため、変更前記録（Step E）はテキスト読取を必須とする（steps-reference E-3 冒頭）
- 検証状況・既知の制約は [TESTING.md](TESTING.md) を参照
- CI（GitHub Actions）が push ごとに参照整合 lint（scripts/lint.py）と hooks スモークテスト（scripts/test-hooks.sh）を実行。ローカルでも同コマンドで実行可
