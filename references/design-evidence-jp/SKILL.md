# 実証デザイン数値基準（日本の実験・公的基準ベース）

数値はすべて出典つき。**「実証」「公的基準」「経験則」を区別して記載** — 経験則は初期値としてAB検証前提。

## 1. タイポグラフィ（確定値）

| 項目 | 基準値 | 根拠の強さ |
|---|---|---|
| 本文サイズ | **16px以上**（管理画面のみ14px許容）。iOS のフォーム input は16px未満で自動ズーム発生→必ず16px | 公的（デジタル庁DADS・Google） |
| シニア主対象 | 本文18px以上 + ズーム阻害禁止（`user-scalable=no` 禁止） | 公的（JIS S 0032 の可読限界×安全率） |
| 注記・キャプション | 12px未満禁止 | 公的（DADS下限運用） |
| 行間 | 本文 `line-height: 1.7`前後（長文1.75）、見出し1.2〜1.4、バナー短文1.3まで | 公的（DADS 1.5倍以上・WCAG 1.4.12） |
| 行長 | **全角20〜35字**（16pxなら max-width 560〜640px）。40字超は読み戻り増加、スマホで字数を稼ぐための縮小は禁止 | 実証（眼球運動実験: 最適20〜29字） |
| 書体 | 画面本文=ゴシック（Noto Sans JP推奨）。明朝は見出し・高級演出のみ・20px以上。**游ゴシックは Windows で細るため本文 weight:500** | 実証+実務定石 |
| UDフォント | 行政・医療・シニア向けは BIZ UD系を第一候補（誤り発見率+5.34%・40代以上で読速+3.3%、モリサワ×自治体 n=294） | 実証（メーカー系、効果は数%） |
| Webフォント | サブセット+WOFF2+`font-display: swap`+ウェイト2種まで（Noto Sans JP フルは1ウェイト1.5〜2MB）。LCP最優先なら見出しのみWebフォント | 実測 |
| ジャンプ率 | 本文16px基準: H3=20 / H2=24 / H1=32 / ヒーロー40〜48px。セール系は3〜4倍、高級・コーポレートは1.5〜2倍 | 経験則（印象操作の定石） |
| 強調 | 下線=リンク専用。太字は1段落1箇所・斜体は日本語で禁止。強調3箇所超は構造化（見出し/箇条書き）へ | 実務確立（内閣官房ガイド系） |
| 文字組み | `line-break: strict`（禁則）+ `text-autospace: normal`（和欧間四分アキ、Baseline 2025）。手動半角スペース挿入は禁止 | 公的（JIS X 4051 / W3C JLREQ） |

## 2. 配色（確定値）

| 項目 | 基準値 | 根拠 |
|---|---|---|
| コントラスト比 | 本文 **4.5:1以上** / 24px以上の見出し・UI部品境界 3:1以上。灰色文字は #767676 より薄くしない | 公的（WCAG 2.x / JIS X 8341-3） |
| 色覚多様性 | グラフ・ステータス色は **CUDO推奨配色セット ver.4** から（赤は朱寄り・緑は青緑寄り）。赤×緑の対比だけで情報を伝えず形状・ラベル併用 | 公的（当事者検証済みパレット。日本人男性の約5%が対象） |
| CTAボタン色 | 「この色が勝つ」は神話（赤+21%等は対比効果）。**基調色の補色系・ページ内で唯一の高彩度色**にし、CTA以外にその色を使わない | 実証の再解釈（AB必須） |
| 和風配色 | 伝統色（NIPPON COLORS等）から基調を選ぶ場合、文字色は必ず 4.5:1 を再検証（通るのは墨・藍系の暗色がほとんど） | 実務 |

## 3. LPレイアウト（実証ベンチマーク）

| 項目 | 基準 | 根拠 |
|---|---|---|
| ファーストビュー | 「誰向け・何が得られる・CTA」の3点を必ず配置。**FVで訴求完結型は CVR 1.64倍**（BtoB 30LP調査）。FV離脱率70%超は作り直し判定 | 実証（WACUL 2019） |
| 主CTA位置 | **FV中央（キャッチ直下）**: CVR 1.10% vs ヘッダー配置0.83%。ヘッダー右上は補助 | 実証（WACUL） |
| CTA文言 | 「行動+ベネフィット+所要時間/無料」の3要素（「3分でわかる資料DL」1.16% vs 「資料請求」0.80%）。ボタン直下に不安払拭1行（「登録無料・1分」でCVR1.5倍の国内例） | 実証 |
| 注視の分布 | 閲覧時間の**57%が1画面目、74%が2画面以内**（NN/g 2018）。重要情報と主CTAは上位2画面、3画面目以降は補足専用 | 実証 |
| LPの長さ | 長さとCVRは相関−0.23（長いほど悪化傾向）。長さは商材の検討コストで決め、説得を长さに頼らない | 実証（WACUL）+ 実務 |
| 構成の型 | 新PASONA / QUEST は「初期仮説」として使い、セクション順はAB検証（型自体のRCT実証は無い） | 経験則と明記 |
| 公開後 | 2週間でスクロール+クリックヒートマップ取得 → **到達率50%を切る位置より下に重要情報を置かない** | 実務判定基準 |
| グリッド | 12カラム / 最大幅1200px / ガター24px / 余白は8px系列（8-16-24…）/ 角丸 R4(〜40px)・R8(41〜119)・R12(120〜) | 公的（DADS）+ 実務標準 |
| ナビ | PCは主要ナビ4〜7項目を表出。ハンバーガーは発見性を下げる（NN/g実証）— 使うなら枠線+「メニュー」ラベル（+20%の事例） | 実証 |
| タップ領域 | 44×44px以上 + 周囲8px | 公的（HIG/WCAG/DADS） |

## 4. インフォグラフィック・グラフ（実証階層）

**第1原則（Cleveland & McGill 1984 のRCT）**: 判断精度は「共通尺度上の位置 > 長さ > 角度 > 面積 > 濃淡」。**比較させたい値は位置/長さ（=棒・折れ線）で符号化**し、面積・角度・色の濃さは最終手段。

| 伝えたいこと | 使う | 根拠 |
|---|---|---|
| 量の比較 | 横棒 | 統計局ガイド+Cleveland-McGill |
| 時系列・傾向 | 折れ線 | 同上 |
| 構成比 | 帯グラフ（時点比較あり）/ 円は**5項目以内・単一時点・過半数を見せる時のみ** | 円の角度比較は不正確（実証） |
| 相関 | 散布図 | 統計局 |

- 軸は原則0起点、単位・出典を必ず記載。縦横比操作・波線省略は印象を歪める（統計局明記）
- **3D・影・グラデ装飾は全面禁止**（3D円は手前が大きく見える=印象操作。統計局も非推奨）
- カテゴリ色は最大6色+強調1色。**強調以外はグレー化する「強調1色設計」を基本形**に（6色は経験則）
- 凡例より直接ラベル優先、グリッド線は薄灰

## 使い方

- 生成前: 該当セクションの基準値をそのまま採用（迷ったら公的>実証>経験則の順に信用）
- レビュー時: この表との差分を指摘として列挙（design-critic の審査観点に使える）
- 経験則マーカーのある項目は cro-jp の手順でAB/前後比較を組む
- 統一骨格・CSS変数は templates/design-principles.md と docs/conventions.md が正本（本スキルは数値の根拠側）
