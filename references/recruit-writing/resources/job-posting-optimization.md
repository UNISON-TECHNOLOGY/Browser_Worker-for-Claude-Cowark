# 求人掲載の最適化リファレンス

求人媒体での掲載効果を最大化するための汎用的なベストプラクティス。
プラットフォーム固有の操作手順はサイトナレッジを参照。

## 効果改善の基本構造

```
エントリー数 = PV数（閲覧） × エントリー率（応募転換）
```

PV数とエントリー率は独立した改善軸。両方を同時に引き上げる。

## PV数の改善

### 1. アイキャッチ画像の4P設計

ターゲット学生が企業のどこに魅力を感じるかを4P観点で選択する。

| P | 観点 | 画像の方向性 | 例 |
|---|------|------------|-----|
| Philosophy | 理念・目的 | キャッチコピー、抽象デザイン、企業カラー | 赤基調で情熱・挑戦を表現 |
| People | 人材・風土 | オフィス写真、社員の笑顔、チームの雰囲気 | 社員の個性が伝わるポートレート |
| Profession | 仕事・事業 | 業務の様子、プロダクト、事業規模 | 海外展開を示す画像、商品写真 |
| Privilege | 特権・待遇 | 初任給、年収、福利厚生の数字 | 「初年度年収○○万円」を大きく |

**選択基準**: 競合他社との差別化ポイントに合致するPを選ぶ。

### 2. アイキャッチのデザイン原則

- 公式HPとトンマナ（カラーテーマ・フォント・メッセージ）を統一する
- 公式HPから画像をそのまま引用するのも効果的
- キャッチコピーは2行以内（スマホ前提）
- 背景色と文字色の重複を避ける

### 3. タイトルの改善

- **内容の具体化**: どんなコンテンツをどのくらいの期間行うか
- **記号で強調**: 冒頭に記号＋魅力1フレーズ
  - `★1次面接確約★`
  - `《社長からのFB付き》`
  - `【期間限定/ES免除エントリー】`
- **実績の訴求**: 市場シェア、受賞歴、売上規模などを端的に

## エントリー率の改善

### 1. 情報設計（何を書くか）

エントリーを判断するために学生が必要とする情報:

| 優先度 | 情報 | 理由 |
|--------|------|------|
| 必須 | 会社概要・強み | イベント一覧から直接流入する学生は企業を知らない |
| 必須 | 募集要項（初任給・勤務地・福利厚生） | 就職先検討の必須事項 |
| 高 | 採用大学実績 | 自分の大学の卒業生がいるか確認する行動 |
| 高 | 当日のスケジュール | 安心感 → 参加意思決定 |
| 中 | こんな方におすすめ / 過去参加者の声 | クチコミが少ない場合の補完 |
| 中 | エントリー後の流れ | 手順が不明だと離脱する |
| 低 | 先輩社員のキャリアパス | 本選考の場合に有効 |

### 2. レイアウト設計（どう書くか）

- **文字量**: 全体で1,200〜1,500字が適切
- **見出し**: 区切り線＋太字で各項目に見出しをつける
- **箇条書き**: 段落ごとに端的に。長文は圧迫感 → 離脱
- **改行**: スマホ画面で詰まって見えないように
- **スマホファースト**: 閲覧の主流はスマートフォン

### 3. 応募ハードルの低減

- 日程を多く設ける（学生の選択肢を広げる）
- 「日程が合わない方向け」の受け皿を用意する
- 事前課題を可能な限り削減する（ES免除も検討）
- 学生の祭事（試験期間等）と募集期間の重複を避ける
- 締切時間は学生の閲覧ピーク（昼休み14-16時、帰宅後22時）に設定

### 4. クチコミの活用

- クチコミ数を増やす → 検索結果の上位表示に寄与
- 内定者・イベント参加者にクチコミ投稿を依頼
- 競合他社のクチコミを分析 → 学生が求める体験を把握 → イベント設計に反映

## recruit-writing スキルとの関係

- **recruit-writing**: 求人コンテンツの「文章」を書くスキル（AIDMA型構成、体験描写、見える化変換）
- **本リファレンス**: 求人掲載の「効果」を最大化する設計ガイド（PV×エントリー率の最適化）
- 両方を組み合わせて使う: 本リファレンスで構造を設計 → recruit-writing で文章を執筆
