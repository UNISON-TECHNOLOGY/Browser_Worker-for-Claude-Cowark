## 営業職

### 職種概要
法人・個人への提案営業、ルート営業、新規開拓など。人材業界では企業担当（RA）として求人獲得・クライアントリレーションを担う。

### 求める人物像

| 項目 | 基準 |
|---|---|
| 学歴 | 不問（高卒OK） |
| 経験 | 不問（未経験歓迎） |
| EQ | ★★★ 最重要 |
| IQ | ★★☆ |
| コアスキル | 行動量、人たらし力、感情コントロール |

### 理想の候補者像
- **体育会系マインド**: 量をこなすことに抵抗がない。テレアポ100件/日を「つらい」ではなく「やるだけ」と捉えられる
- **高EQ**: 表面的な愛想の良さではなく、相手の感情を読み取り、場の空気をコントロールできる力。共感力が高い
- **表現力がある**: 話がうまい、ではなく「相手に伝わる言葉を選べる」。プレゼンだけでなく、雑談・アイスブレイクも得意
- **人を動かすことへの抵抗感がない**: 説得・交渉・クロージングを「操作」ではなく「最適解への導き」と捉えられる。相手の意思決定を後押しすることに罪悪感を持たない
- **打たれ強い**: 断られても切り替えが早い。失敗を引きずらず次のアクションに移れる

### 面接での見極めポイント
- 「断られた時どうする？」→ 切り替えスピード
- 「人を説得した経験は？」→ 相手視点の有無
- 過去のスポーツ・部活動経験 → 行動量耐性
- 初対面の面接官との距離の詰め方 → 天性の人たらし度

### 求人キーワード
`未経験OK` `高卒歓迎` `インセンティブ` `成果主義` `体育会系歓迎` `行動力` `人と接する仕事` `稼ぎたい`

## マーケター

### 職種概要
デジタルマーケティング、コンテンツ企画、CRM、データ分析、ブランド戦略など。人材業界では求職者集客、採用マーケティング支援を担う。

### 求める人物像

| 項目 | 基準 |
|---|---|
| 学歴 | MARCH以上（高学歴） |
| 経験 | 業務での工夫・改善経験必須 |
| IQ | ★★★ 最重要 |
| EQ | ★★☆ |
| コアスキル | ロジカルシンキング × 発想力、垂直・水平思考 |

### 理想の候補者像
- **垂直思考 × 水平思考の両刀**: ロジカルに深く掘り下げる分析力と、横に広げる発想力の両方を持つ。「なぜ？」と「もし〜だったら？」を自在に切り替えられる
- **理想は理系脳×文系感覚**: 理系的なデータ・論理をベースに持ちながら、文系的な「人の感情」「言葉の力」を理解できる。あるいは文系だがロジックで感覚を紐解ける
- **努力家**: 才能ではなく努力で結果を出してきた人。PDCAを自然に回せる。学び続ける習慣がある
- **工夫を語れる**: 「やりました」ではなく「こう考えて、こう工夫して、こう改善した」を具体的に語れる。再現性のある思考プロセスを持っている
- **人間に対する深い関心**: 人間が大好きか、あるいは人間をよく観察している。ターゲットの心理を深く理解し、インサイトを掴める。「なぜ人はこう行動するのか」に好奇心がある

### 面接での見極めポイント
- 「前職でどんな工夫をした？」→ 思考プロセスの言語化力
- 「このデータからどんな仮説を立てる？」→ 垂直×水平思考
- 「ターゲットの気持ちを想像して」→ 人間理解の深さ
- 大学での専攻・研究テーマ → 論理的な思考訓練の土台
- 趣味・読書歴 → 知的好奇心の幅と深さ

### 求人キーワード
`マーケティング` `データ分析` `企画` `戦略` `ロジカル` `仮説検証` `PDCA` `広告運用` `CRM` `グロース`

## インフラエンジニア

### 職種概要
サーバー、ネットワーク、クラウド（AWS/Azure/GCP）の設計・構築・運用。人材業界では自社システム基盤のほか、クライアント企業への技術者派遣・SESも含む。

### 求める人物像

| 項目 | 基準 |
|---|---|
| 学歴 | 高学歴だと嬉しい（必須ではない） |
| 経験 | 未経験可（ただし資格学習歴は見る） |
| IQ | ★★★ |
| EQ | ★★☆ |
| コアスキル | 自学力、粘り強さ、コミュニケーション |

### 理想の候補者像
- **勉強好き**: 技術の変化が激しいインフラ領域で、自発的に学び続けられる。資格取得（CCNA、AWS SAA、LPIC等）に意欲的。休日に自宅ラボを組むようなタイプ
- **努力家**: 一つのことをコツコツ積み上げられる。華やかさはないが着実に成長する。障害対応で深夜になっても粘れる根性
- **コミュニケーション能力**: インフラは「縁の下の力持ち」だが、開発チームやクライアントとの調整が必須。技術を非技術者に説明できる力。「何が起きているか」を正確に伝えられる
- **慎重さ×大胆さ**: 本番環境への変更は慎重に。しかし技術選定や新技術の導入には大胆に挑戦できるバランス感覚

### 面接での見極めポイント
- 「最近何を勉強した？」→ 自学力の有無
- 「障害が起きたらどう対処する？」→ 論理的切り分け能力
- 資格取得歴・学習計画 → 努力の可視化
- 「チームでの役割は？」→ コミュニケーションスタイル

### 求人キーワード
`AWS` `Azure` `インフラ` `ネットワーク` `Linux` `クラウド` `資格支援` `研修充実` `リモート` `運用から構築へ`

## 上流SE / PM（プロジェクトマネージャー）

### 職種概要
要件定義、基本設計、プロジェクト管理。クライアントの業務課題をシステムで解決する上流工程のスペシャリスト。

### 求める人物像

| 項目 | 基準 |
|---|---|
| 学歴 | 問わない（実力主義） |
| 経験 | **経験者限定**（要件定義・PM経験必須） |
| IQ | ★★★ |
| EQ | ★★★ |
| コアスキル | ビジネス理解 × 技術知見 × 推進力 |

### 理想の候補者像
- **発想力**: 「クライアントが言っていること」と「本当に必要なこと」の差分を見抜ける。言われたものを作るのではなく、あるべき姿を提案できる
- **ビジネス理解**: 技術だけでなく、クライアントの事業構造・収益モデル・業界特性を理解している。「なぜこのシステムが必要か」をビジネス言語で語れる
- **要件定義力**: 曖昧な要望を構造化し、実現可能な要件に落とし込める。ステークホルダー間の利害を調整し、合意形成までリードできる
- **推進力**: 計画を立てるだけでなく、遅延・課題・リスクに対して自ら動いて解決する。チームを巻き込み、プロジェクトを前に進められる牽引力
- **コミュニケーション**: 経営層にはビジネス言語で、開発チームには技術言語で、ユーザーには平易な言葉で。相手に合わせた翻訳能力

### 面接での見極めポイント
- 「最も困難だったプロジェクトは？」→ 推進力・問題解決力
- 「要件が曖昧な時どうする？」→ 構造化能力
- 「ビジネス側と技術側の対立をどう解決した？」→ 調整力
- 過去のプロジェクト規模・人数・金額 → 経験の深さ
- **経験者限定のため、実績ベースでの評価が最重要**

### 求人キーワード
`PM` `プロジェクトマネージャー` `要件定義` `上流工程` `PL` `基本設計` `マネジメント` `経験者優遇` `年収600万以上`
