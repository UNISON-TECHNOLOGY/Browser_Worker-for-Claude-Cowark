# UX心理学の根拠層（Laws of UX 系の再構成）

デザイン判断の「なぜ」を説明するときに引く原則集。改善提案レポートの根拠付けに使う。

| 原則 | 内容 | LP設計への適用 |
|---|---|---|
| ヒックの法則 | 選択肢が増えるほど決定は遅くなる | CTAは1種類。ナビは削る。料金プランは3つまで |
| フィッツの法則 | 大きく近いものほど押しやすい | 主CTAは大きく・親指の届く位置に（モバイル下部） |
| ミラーの法則 | 短期記憶は7±2チャンク | 箇条書きは3〜5項目。特長は3つに絞る |
| 美的ユーザビリティ効果 | 美しいものは使いやすいと知覚される | 整った余白・整列は信頼に直結（雑な整列は内容の信頼も毀損） |
| ヤコブの法則 | ユーザーは他サイトの慣習を期待する | 独創的なナビ/フォームより慣習に従う。ロゴは左上=トップへ |
| ピークエンドの法則 | 体験はピークと終わりで記憶される | フォーム完了画面・サンクスページを手抜きしない |
| フォン・レストルフ効果 | 異質なものだけが記憶に残る | だからアクセント色はCTA専用（全部目立つ=何も目立たない） |
| 系列位置効果 | 最初と最後が記憶に残る | 特長リストの1番目と最後に最強の訴求を置く |
| 認知負荷 | 処理容量を超えると離脱する | 1画面1メッセージ。専門用語は翻訳。計算させない（「月あたり◯円」を明記） |
| ゴール勾配効果 | ゴールが近いほど加速する | 複数ステップのフォームは進捗バーを見せる（残り1問と示す） |

使い方: レポートの改善提案には可能な限り原則名を添える（例: 「CTA を1種に統合 — ヒックの法則」）。根拠のない提案は design-critic が REVISE 対象にできる。
