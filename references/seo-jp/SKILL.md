# 日本語SEO/AEO スキル
**対象領域: 採用オウンドメディア・BtoB/BtoC サービスサイト（日本語圏）**

## 設計原則（優先順）

1. **検索意図が先、キーワードは後** — 1ページ=1意図。「知りたい（Know）/行きたい（Go）/やりたい（Do）/買いたい（Buy）」を最初に確定し、意図が混ざったら分割する
2. **E-E-A-T の実装は具体で** — 経験談（一次情報・実測値・事例）、著者情報（実名・肩書・プロフィールリンク）、更新日の明示。「専門家が語る」ではなく専門の中身を書く
3. **AEO/GEO（AI検索対応）** — 冒頭150字で結論を完結に述べる（AIが引用する単位）、FAQ形式の節を設ける、定義文は「AとはBである」の形で独立させる、表・箇条書きで構造化（LLMが抽出しやすい）
4. **日本語特有** — 表記ゆれの統一（例: 「エンジニア採用」「エンジニア 採用」は検索上別物として調査）、共起語は上位10記事の頻出語から拾う、タイトルは32字以内・重要語を前方に

## 記事構成の型

```
タイトル（32字・数字か具体語を入れる）
├─ リード（150字で結論 + この記事でわかること）
├─ H2 × 3〜6（検索意図の分解。H2だけ読めば全体がわかる見出し文に）
│   └─ H3（各論。1節=1トピック、300〜600字）
├─ FAQ（よくある質問 3〜5問。AEO対策の主戦場）
└─ まとめ（結論の再掲 + 次のアクション1つ）
```

## 診断チェックリスト（既存ページ）

- [ ] title / meta description / H1 が検索意図と一致しているか（title 32字・description 120字目安）
- [ ] 見出し階層が論理的か（H2飛ばしのH3、装飾目的の見出しはNG）
- [ ] 結論が冒頭にあるか（結論後置は AI にも人にも拾われない)
- [ ] 内部リンク: 関連記事へ3本以上・アンカーテキストが具体的か
- [ ] 構造化データ（FAQ / Article / パンくず）の有無 — 実装は web-design と連携
- [ ] 更新日が古くないか（1年超は鮮度リライト候補）

## 禁止・注意

- 検索ボリュームだけでテーマを選ばない（採用サイトなら「応募につながる意図」を優先）
- 生成AI丸出しの網羅記事は評価されない — 一次情報（自社データ・実務者の声）を必ず1つ以上入れる
- 医療・金融・法律に踏み込む表現は YMYL — 断定を避け出典を付ける。広告表現は ad-compliance-jp を通す
- 診断結果をレポート化するときは logical-writing / conventions.md に従う
