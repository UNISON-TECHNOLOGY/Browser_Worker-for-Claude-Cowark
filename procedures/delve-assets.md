# /素材探し — 画像素材の探索・取り込み（delve-assets）

ストック素材サイトを巡回し、要件に合う画像素材を「候補提示 → 承認 → ライセンス証跡 → 取り込み → 出典台帳」の流れで調達する。

## 手順

### 1. 要件の確定

用途（LPヒーロー/バナー/SNS投稿/資料）、形状（横長/正方形/背景透過）、テイスト、人物有無、商用利用の要否を依頼文から読み取る。不明点が2つ以上あれば質問ツールで確定する。

### 2. 巡回先の選定（ライセンスが明確なサイトを正とする）

**初回のみ**: `knowledge/config/accounts.md` を確認し、記録がなければ**どの素材サイトのアカウント（無料/有料）を持っているか**をユーザーに質問して記録する。有料アカウントがあるサイトを優先巡回先にする（DL制限・クレジット表記条件・解像度が変わるため。2回目以降はこの記録を読むだけで質問しない）。

`knowledge/sites/` の素材サイトナレッジを確認（あればフェーズ②で最短巡回）。初回はフェーズ①としてマッピングする（/タスク開始 経由。ただし探索・閲覧のみなら読み取りフリー）。

| サイト | 向き | 固有ルール（初回マッピング時に実数を確認して knowledge/sites/<site>/index.md に記録） |
|---|---|---|
| Pngtree | 背景透過PNG・イラスト | 無料枠はクレジット表記必須+日次DL制限。誤クリック誘導UIが多く正規DLボタンの位置をナレッジ化する価値大 |
| Unsplash / Pexels | 写真（クレジット不要） | 商用可・表記不要だが人物写真のモデルリリースは個別確認 |
| photoAC / イラストAC | 日本向け写真・イラスト | 無料会員はDL回数制限+サイズ制限 |

Bing/Google 画像検索は「発見」用途のみ。**権利不明の拾い画像は取り込まない**（見つけた画像の出典サイトへ辿って正規入手）。

### 3. 探索と候補提示（Step H 相当）

1. 検索・絞り込みで候補を集め、**上位5〜10点**をサムネイル一覧（タイトル/サイト/ライセンス種別/URL）でユーザーに提示する
2. ユーザーが選定 → 選ばれたものだけ取り込みへ

### 4. ライセンス証跡 CP（取り込み前・必須）

ダウンロード前に素材ごとに CP を照合する（証跡なしで取り込み禁止 — steps-reference E-3 準拠）:

- [ ] **商用利用可**か（ライセンスページの該当記述）
- [ ] **クレジット表記の要否**（要なら表記文をこの時点で確定）
- [ ] 人物写真は**モデルリリース**の有無
- 証跡 = ライセンス表記のスクリーンショット or 記述ページURL。**取り込みと同時に出典台帳へ記録**

### 5. 取り込み

- blob/画像URL の fetch を第一とし、ダウンロードボタン経由の場合は**専用DLフォルダ**（Downloads 本体は接続しない — delve-imagegen の取り込み表と同じ原則）を使う。fetch → `knowledge/assets/<用途>/<name>.<ext>` に保存（/タスク開始 のゲート下で実施）
- 加工（リサイズ・切り抜き）が必要なら Pillow 規範に従う（**人物写真の印象改変は禁止**）

### 6. 出典台帳（正本: knowledge/assets/sources.md）

1素材1行で追記: `| 日付 | ファイル | 入手元URL | ライセンス種別 | クレジット表記（要否と文言） | 証跡 |`
成果物（LP/バナー/投稿）で使う際は、この台帳のクレジット要否に従うこと。

### 7. 完了

取り込み結果（点数・保存先・クレジット要否の注意）を報告し、session-log に1行記録。

## 注意

- DL制限・有料プラン誘導ページに入ると Money Watch が発火する（設計どおり停止 → 復帰手順に従う）。**素材サイトの有料プラン購入・課金は AI は行わない**
- 同一要件の探索が2回目以降なら、前回の sources.md と assets/ を先に確認（再DLの無駄を省く）
