添付された広告クリエイティブを起点に、メッセージ一致した LP モックアップを生成してください。補足情報: $ARGUMENTS

## 手順

1. **広告の解析**（画像を直接読む）: 添付画像から以下を抽出して `knowledge/styles/ad-<name>.json` に記録
   - コピー: メインコピー・サブコピー・CTA文言（一言一句そのまま）
   - ビジュアルトーン: 主要色（可能な限りHEX推定・「目視推定」と明記）・フォントの雰囲気（丸ゴ/角ゴ/明朝・太さ）・写真/イラストの別と被写体
   - 訴求タイプ: ベネフィット型 / 実績型 / 共感型 / オファー型（割引等）
   - ターゲット推定: 誰向けか（画像内の人物・文言から）
2. **メッセージ一致の設計**: 広告→LPの連続性を最優先する
   - LPヒーローの見出しは広告のメインコピーと同一または直接の続きにする（言い換え禁止 — 広告をクリックした人が「同じ話だ」と0.5秒で分かること）
   - 広告のCTA文言とLPの主CTA文言を一致させる
   - 広告の主要色をLPのアクセント色に採用（references/web-design の 60-30-10 に配置）
3. **LP構成の組み立て**: references/web-design/resources/lp-cro.md の標準骨格に、訴求タイプ別の重点を反映
   - 実績型 → 証拠セクションを前倒し / 共感型 → 課題提起を厚く / オファー型 → 期限・条件を明示
   - 本文コピーが必要な箇所は該当ライティングスキル（sales-writing等）を適用。捏造禁止（広告にない実績数値は `[要確認]`）
4. **生成と検証**: design-artisan に委譲（広告解析JSON・抽出コピー・構成案の絶対パスを渡す）→ design-critic の審査ループ（delve-improve と同じ、最大2周）→ アーティファクト発行
5. **メッセージ一致チェック**を最後に自己実行: 広告とモックアップを並べ、コピー・色・トーンの一致点と意図的な差異を表で報告
6. **法規チェック（必須）**: LP本文の最終稿を references/ad-compliance-jp/SKILL.md で検査（景表法の優良/有利誤認・薬機法・打消し表示）。指摘があれば修正してから納品

## 出力

- 広告解析: `knowledge/styles/ad-<name>.json`
- モックアップ: `knowledge/mockups/adlp-<name>-<date>.html` + アーティファクト共有URL
- 一致チェック表つきの簡易レポート（チャット内でよい。フルレポートは求められたら）
- **Claude Design 同期（任意・DesignSync ツールがある環境）**: ユーザーが「Design に同期して」「デザインシステムに登録して」と求めたら、DesignSync で claude.ai/design のプロジェクトに push する。手順: list_projects で対象確認（なければ create_project）→ 書き込むパスの計画を提示してユーザー承認（finalize_plan）→ write_files。**承認された範囲外の書き込み・一括置き換えは禁止**（1コンポーネントずつ増分同期）。同期先URLを knowledge/artifacts-index.md に記録

### 動く演出部品（任意）

ヒーローの人物や CTA 横のオブジェクトを動かしたい場合は、docs/media-pipeline.md の「LPの動く人物/オブジェクト」ライン（GB生成→chromakey→アルファWebM→`<video>` 重ね）を使う。演出は本体を侵食しない範囲で。

## 注意

- 添付画像は knowledge/assets/ に保存（implementation.md の画像規範に従う）
- 広告画像自体をLPに流用する場合は、権利がユーザー側にあることを確認してから
- 他社の広告を渡された場合は「参考トーン」としてのみ使い、コピーの転用はしない（自社広告のときだけコピー一致設計を行う）
