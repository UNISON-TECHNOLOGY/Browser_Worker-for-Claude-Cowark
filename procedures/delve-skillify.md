直近の会話・作業（$ARGUMENTS）から、昇格すべき知を抽出して定着させてください。

## 3種の定着先（振り分け基準）

| 種類 | 判定 | 保存先 |
|---|---|---|
| **手順（スキル）** | 「同じやり方で再実行できる方法」— ユーザーが教えた手順、2回以上繰り返した作業パターン、試行錯誤の末に確立したやり方 | ワークスペースの `.claude/skills/<name>/SKILL.md` |
| **暗黙知（ナレッジ）** | 「調べても出てこない事実」— 業界事情、社内ルール、取引先の癖、この人しか知らない背景・判断基準 | `knowledge/tacit/<topic>.md` |
| **好み（既存）** | 成果物への OK/NG | `knowledge/feedback/lessons.md`（/メモリ保存 の管轄） |

## スキル化の手順

1. 手順を再実行可能な形に整理する（前提 → ステップ → 検証方法 → 失敗時の対処）
2. `.claude/skills/<kebab-name>/SKILL.md` として書く。frontmatter 必須:
   - `name`: ケバブケース
   - `description`: **「Use when + ユーザーの言い方の例」形式**（自動発火の生命線。本プラグインのコマンド群と同じ作法）。**言い方の例は必ず日本語で書く**（ユーザーは日本語で話しかけるため。英語の例だけだと発火しない）
3. 具体値（URL・ID・数値）は本文に残してよい（ワークスペース内に閉じるため）。ただし認証情報は絶対に書かない
4. 既存スキル（プラグイン同梱+ワークスペース）と重複しないか確認。重複するなら新規作成せず既存を更新
5. 作成後「スキル化しました: <name> — 次からは『◯◯して』で発火します」と1行報告

## ナレッジ化の手順

1. `knowledge/tacit/` 配下にトピック単位で記録（1ファイル1トピック、150行超えたら分割）
2. 形式: 事実 → 出所（誰から/いつ）→ 適用場面。推測と事実を混ぜない
3. 既存ファイルがあれば追記・更新（矛盾したら新しい方を正とし、古い記述に取り消し線）
4. 記録後「ナレッジ化しました: <topic>」と1行報告

## 自動発火の合図（この状況を見たら提案ではなく実行する）

- ユーザーが手順を口頭で説明した（「うちではこうやってる」「この順でやって」）
- 同じ種類のタスクで前回と同じ手直しをした（2回目=パターン）
- 業界・社内・媒体固有の事実が会話に出た（「この媒体は金曜に審査が止まる」等）
- 明示指示（「覚えて」「スキルにして」「次からこれで」）

## 注意

- スキル・ナレッジはワークスペース産物（フォルダ接続していないと消える — 未接続なら記録前にその旨を警告する）
- 記録は静かに素早く（長い確認をユーザーに求めない。1行報告のみ）
- 個人情報（候補者名等）はナレッジに書かない。役割・属性に抽象化する
