# /ワーク追加 — 媒体・ワークの登録 + 初期マッピング + 動的コマンド生成（delve-add-work）

新しい媒体（ワンキャリア・doda・Green 等の求人媒体、SNSアカウント、広告アカウント、自社サイト）を1コマンドで運用に載せる。**登録して終わりではなく、その場で初期マッピング（フェーズ①）まで行い、専用コマンドを生やす**のが本体。

## 手順

### 1. ヒアリング（不足分のみ質問）

- 媒体名 / 管理画面URL（引数で貰えていれば聞かない）
- 種別: 求人媒体 / SNS / 広告 / オウンドメディア
- 管理したい残数・期限（スカウトチケット、契約更新日など。不明なら「マッピング中に見つけたら拾う」でよい）
- ログインはユーザーが済ませる（既存原則: 認証情報は扱わない。未ログインならここで依頼して待つ）

### 2. 台帳登録

`knowledge/media/registry.yaml`（正本・スキーマは procedures/delve-media.md）に1エントリ追記。id は英語ケバブケース（例: onecareer, doda）。

### 3. 初期マッピング（フェーズ① First Delve — 読み取り専用）

タスク開始手順（procedures/delve-start.md）に従い、**変更操作なしの探索**として実行:

1. 管理画面のトップから主要ナビを snapshot でたどる（読み取りのみ。設定変更・送信は一切しない）
2. `knowledge/sites/<id>/index.md` を生成 — 基本情報 / ナビ構造 / 主要ページへのリンク / 発見した残数・期限の場所（watch 候補）
3. 主要業務ページ（求人一覧・スカウト画面・分析画面など）は見つけた範囲でページ単位ファイルに分割（150行ルール。D-2 準拠）
4. 探索で見つけた quota / renew_date を registry.yaml に反映
5. bot検知・2FA・CAPTCHA に遭遇したら即中断してユーザーへ（登録と動的コマンド生成は完了させる）

### 4. 動的コマンド生成

ワークスペースの `.claude/commands/<媒体名>.md` を生成（プラグイン本体は増やさない）:

```markdown
---
description: <媒体名> — この媒体専用パック。Use when ユーザーが「<媒体名>で◯◯して」と依頼したとき（求人更新/スカウト/状況確認等）。
argument-hint: <要望>
---
knowledge/media/registry.yaml の該当媒体と knowledge/sites/<id>/ のナレッジを Read し、
プラグインの procedures/delve-media.md（タスク分解表）に従って実行してください。引数: $ARGUMENTS
```

**親パックの判定（マトリョーシカ）** — 生成する専用コマンドは必ず親の配下に置き、親の共通フロー（ゲート・部品・規範）をポインタで継承する。子に手順を複製しない:

| 媒体の種類 | 親 | 子コマンドの参照先 | 命名 |
|---|---|---|---|
| SNS | /SNS運用 | procedures/delve-sns-<媒体>.md | `<媒体名>運用`（/X運用） |
| 広告 | /広告 | procedures/delve-ads.md（媒体固定・§0質問2スキップ） | `<媒体名>広告`（/Google広告） |
| 求人 | /媒体管理 | procedures/delve-media.md（媒体固定） | `<媒体名>`（/doda） |
| 自社サイト | /Webサイト | procedures/delve-website.md（サイト固定） | `<サイト名>` |
| **どれにも該当しない完全別ワーク** | **「その他ワーク」= 非表示の親**（メニューコマンドは作らない） | docs/parts/index.md のタスク5型 + procedures/delve-start.md のゲート（共通フローは直書きせずポインタ） | `<ワーク名>` |

registry.yaml のエントリに `parent: sns|ads|recruit|website|other` を必ず記録する（ダッシュボード・/カスタマイズ list のグルーピングに使う）。/セットアップ の媒体選択（SNS・広告・求人）も同じ表で生成する。アカウント単位で分けたい場合はアカウント名コマンド（例: /IT就活大学）でもよい。

### 5. 締め

- 定常ループの提案: 「毎朝ステータス巡回に入れますか？」（承諾なら /カスタマイズ のタスク登録で）
- 報告（定型 — この言い回しを使う）: 「**/<媒体名> コマンドを追加しました。次回以降 /<媒体名> でタスクを入れてもらえれば、地図（ナレッジ）を厚くしながら作業速度を改善していきます**」+ 登録内容 / 生成したナレッジファイル / マッピングで見つけた注意点。※コマンドの有効化は次セッションからな点も一言添える
- session-log に1行記録

## 削除

「<媒体名> を外して」→ registry.yaml のエントリ、`.claude/commands/<媒体名>.md`、loops.yaml の関連行を削除（knowledge/sites/<id>/ の地図は資産として残す。消すのはユーザー明示時のみ）。
