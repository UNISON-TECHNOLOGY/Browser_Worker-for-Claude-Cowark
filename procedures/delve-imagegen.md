# /画像生成 — 生成AIによる画像作成と取り込み（delve-imagegen）

ユーザーのログイン済みブラウザで Gemini（第一）/ ChatGPT（代替）の画像生成を操作し、生成画像をワークスペースに取り込む。2026-07-23 に Gemini 経路で実機検証済み。

## 前提と節度

- ログインは人間（既存原則どおり）。アカウントのプラン・生成枠はユーザーのものを消費するため、**枚数の当てを先に合意**する（「◯枚生成して選ぶ」形）
- Web UI の自動操作は各社規約上グレー。**たまに数枚**の運用に留め、量産が必要になったら正規 API（Gemini API / gpt-image-1）への切替をユーザーに提案する（I-5a と同じ思想）
- bot 検知・CAPTCHA に遭遇したら即中断して人間へ

## 手順

### 1. プロンプト設計

用途（バナー背景/挿絵/SNS画像）・形状・テイスト・文字入れの有無を確定し、生成プロンプトを提示して承認を得る。文字入れは生成AIが苦手なので原則「画像は無地・文字は後入れ」を推奨。

### 2. 生成（/タスク開始 のゲート下）

1. `knowledge/sites/gemini/`（または chatgpt/）のナレッジを確認。初回はフェーズ①で UI をマッピング（入力欄・生成ボタン・画像の出現位置・ダウンロード導線）
2. プロンプト入力 → 生成待ち（wait系で画像出現を待機）→ 結果をスクショでユーザーに提示
3. リテイクは修正指示を添えて再生成（合意した枚数の範囲内）

### 3. 取り込み（2026-07-23 実測知見あり）

採用が決まった画像のみ取り込む。**用途でルートを選ぶ**:

| 用途 | ルート | 備考 |
|---|---|---|
| 案出し・雰囲気確認 | スクショ切り出し | 手軽だが**画面表示解像度どまり**（実測 約565×330px）+ スクロール位置でフレーミングがずれる。本番素材には使わない |
| 本番素材（フル解像度） | (a) 保存ボタン → Downloads 回収 | **Downloads フォルダのワークスペース接続が前提**。未接続ならユーザーに接続を依頼（次回から完結） |
| 〃 | (b) blob/URL を JS fetch → base64 保存 | UI に URL が露出していれば最速。`knowledge/assets/generated/<name>.<ext>` へ |
| 量産・定常 | (c) 正規 API（Gemini API / gpt-image-1） | UI 自動操作から切替をユーザーに提案 |

### 4. 台帳記録（/素材探し と同じ正本: knowledge/assets/sources.md）

`| 日付 | ファイル | 生成元（Gemini/ChatGPT）| プロンプト要約 | 利用条件メモ | 証跡 |` を1行追記。
生成画像の商用利用条件・透かし（SynthID 等）はアカウントのプラン依存のため、初回にユーザーへ確認して index.md に記録する。

### 5. 完了

保存先・枚数・残る注意（文字入れ・権利面）を報告し、session-log に1行記録。

## 注意

- **人物の実在感が強い画像・実在人物風の生成は依頼されても作らない**（肖像・誤認リスク）。人物が必要なら /素材探し のモデルリリース付きストックを使う
- 有料プランへのアップグレード誘導ページは Money Watch / URL Guard の対象（AI は課金しない）
- 生成物を広告・求人に使う場合は ad-compliance-jp のチェックを通す
