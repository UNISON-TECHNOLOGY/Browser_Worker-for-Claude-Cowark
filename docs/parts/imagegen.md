# 部品: 画像生成（旧 /画像生成。動画生成 §2b を含む）

ユーザーのログイン済みブラウザで Gemini（第一）/ ChatGPT（代替）の画像生成を操作し、生成画像をワークスペースに取り込む。2026-07-23 に Gemini 経路で実機検証済み。

## 前提と節度

- ログインは人間（既存原則どおり）。アカウントのプラン・生成枠はユーザーのものを消費するため、**枚数の当てを先に合意**する（「◯枚生成して選ぶ」形）
- Web UI の自動操作は各社規約上グレー。**たまに数枚**の運用に留め、量産が必要になったら正規 API（Gemini API / gpt-image-1）への切替をユーザーに提案する（I-5a と同じ思想）
- bot 検知・CAPTCHA に遭遇したら即中断して人間へ

## 手順

### 0. 生成サービスの確認（初回のみ）

`knowledge/config/accounts.md`（/セットアップ で記録済みのはず）を確認し、記録がなければ**どのサービスのアカウントを持っているか（Gemini / ChatGPT / 両方）とプラン（無料/有料）をユーザーに質問**して記録する。勝手に Gemini と決め打ちしない。
- 両方あれば用途で使い分けを提案（写実・GB指定の忠実さ→Gemini 実績あり / それ以外は好みで）
- 有料プランなら生成枠・解像度・商用条件をあわせて確認し記録（2回目以降はこの記録を読むだけで質問しない）

### 1. プロンプト設計

用途（バナー背景/挿絵/SNS画像）・形状・テイスト・文字入れの有無を確定し、生成プロンプトを提示して承認を得る。文字入れは生成AIが苦手なので原則「画像は無地・文字は後入れ」を推奨。

### 2. 生成（/タスク開始 のゲート下）

1. `knowledge/sites/gemini/`（または chatgpt/）のナレッジを確認。初回はフェーズ①で UI をマッピング（入力欄・生成ボタン・画像の出現位置・ダウンロード導線）
2. プロンプト入力 → 生成待ち（wait系で画像出現を待機）→ 結果をスクショでユーザーに提示
3. リテイクは修正指示を添えて再生成（合意した枚数の範囲内）

### 2b. 動画生成（2026-07-23 Gemini 経由で実機検証済み）

Gemini の動画生成（Veo 系）も画像と同じ UI 操作の型で使える。追加の注意:

- 動画は生成時間が長い（分単位）。wait 系で完了を待機し、途中でセッションを離れない
- 生成枠の消費が画像より重いプランが多い。**本数の当てを先に合意**（原則1〜2本→採用判断）
- プレビューは画面録画やスクショで確認し、採用が決まった本体のみ §3 のルート（専用DLフォルダ / blob fetch）で取り込む
- 後段（WebM 圧縮・アルファ化・LP 埋め込み）は docs/media-pipeline.md の ffmpeg レシピへ接続

### 3. 取り込み（2026-07-23 実測知見あり）

採用が決まった画像のみ取り込む。**用途でルートを選ぶ**:

| 用途 | ルート | 備考 |
|---|---|---|
| 案出し・雰囲気確認 | スクショ切り出し | 手軽だが**画面表示解像度どまり**（実測 約565×330px）+ スクロール位置でフレーミングがずれる。本番素材には使わない |
| 本番素材（フル解像度） | (a) 保存ボタン → 専用DLフォルダ回収 | **Downloads 本体は接続しない**（個人書類が丸ごと AI から見える）。Chrome の保存先を専用フォルダ（例: CoworkDrop）に変えるか「保存場所を確認」を ON にして都度そこへ保存し、その専用フォルダだけを接続してもらう。**Downloads→チャット手動アップロード経由はレンダリング（加工処理）に失敗する（2026-07-23 実測）ため本番素材のルートにしない** |
| 〃 | (b) blob/URL を JS fetch → base64 保存 | UI に URL が露出していれば最速。`knowledge/assets/generated/<name>.<ext>` へ |
| 量産・定常 | (c) 正規 API（Gemini API / gpt-image-1） | UI 自動操作から切替をユーザーに提案 |

### 3b. バナー合成（任意 — 2026-07-23 実機検証済みの型）

取り込んだ画像にコピーを焼き込んでバナー化する場合は `templates/banner-compose.py` を使う（16:9センタートリミング→1280×720→下部グラデーション→キャッチ+サブ+バッジ焼き込み、Noto Sans CJK）:

1. コピー文言は copywriting スキルで作成し、公開向けは **ad-compliance-jp チェックを通してから**焼き込む
2. `python3 templates/banner-compose.py <入力> <出力.png> --headline "…" --sub "…" --badge "…"`
3. コピー案リスト×画像でループすれば量産可（/定常タスク に載せられる）

**背景除去 = グリーンバック生成 + クロマキーが第一手（2026-07-23 実機検証済み）**: 切り抜き素材が欲しいときは、生成プロンプトに **「背景は #00FF00 のグリーンバック一色」を含めて生成**し、`templates/chromakey.py` で透過PNG化する（緑の優勢度→連続アルファ+スピル抑制。MLモデル不要・決定論的・数秒）。Gemini はグリーンバック指定に忠実で再現性が高い。
`python3 templates/chromakey.py <入力> <出力.png> --bg "#f5f2ea"`（--bg で合成確認画像も出力）
※ rembg（MLモデル方式）はモデル本体のDLがサンドボックスでブロックされるため使わない。普通の背景で生成済みの画像を後から抜きたい場合のみ、Canva / Claude Design 側の背景除去に寄せる。

### 4. 台帳記録（asset-collect 部品と同じ正本: knowledge/assets/sources.md）

`| 日付 | ファイル | 生成元（Gemini/ChatGPT）| プロンプト要約 | 利用条件メモ | 証跡 |` を1行追記。
生成画像の商用利用条件・透かし（SynthID 等）はアカウントのプラン依存のため、初回にユーザーへ確認して index.md に記録する。

### 5. 完了

保存先・枚数・残る注意（文字入れ・権利面）を報告し、session-log に1行記録。

## パイプライン全体図

切り抜き・バナー化・透過WebM化・LP埋め込みまでの部品一覧と ffmpeg レシピは **docs/media-pipeline.md（正本）** を参照。

## 注意

- **人物の実在感が強い画像・実在人物風の生成は依頼されても作らない**（肖像・誤認リスク）。人物が必要なら asset-collect 部品のモデルリリース付きストックを使う
- 有料プランへのアップグレード誘導ページは Money Watch / URL Guard の対象（AI は課金しない）
- 生成物を広告・求人に使う場合は ad-compliance-jp のチェックを通す
