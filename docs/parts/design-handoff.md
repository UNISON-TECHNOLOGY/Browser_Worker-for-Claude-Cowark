# 部品: Claude Design 人間編集ハンドオフ（design-handoff — 「キャンバに送って仕上げます」）

> 成果物の最終調整は人間がやる。AIは完成品を Claude Design のキャンバスに載せ、
> タブを開いた状態で人間に手渡すところまでが仕事。編集そのものには介入しない。
> 実証: 2026-07-24 Cowork(cloud) — PPTX添付→プロジェクト生成までUI経由で成立
> （プロジェクト例: claude.ai/design/p/d6e057b1-…）。

## いつ使う（トリガー）

- 成果物（スライド/PPTX・バナー・LP モックアップ・カルーセル等の**見た目もの**）が
  critic 監査 PASS で完成した直後、ユーザーに1行で聞く:
  **「Claude Design に送って手直ししますか？」**
- ユーザーが「Designに送って」「キャンバスで直したい」と言ったときは質問なしで直行
- **送らないもの**: テキストのみの成果物（記事・投稿文）、未監査のドラフト、
  個人情報を含む素材（notion-publish-rules.md の切り抜き規約を通過していないもの）

## 前提条件（順に確認。満たさなければ中止して案内）

1. **経路の選択**: DesignSync ツールの認可をまず1回だけ試す（`list_projects`）。
   - 認可あり → docs/parts/design-sync.md の増分同期ループを使う（API直。そちらが正）
   - 認可なし（Cowork cloud は通らない: 実測 2026-07-24）→ 本部品の **UIルート** へ
2. UIルート: ユーザーの Chrome で claude.ai/design を**ログイン済みで開ける**こと
   （開けなければ「claude.ai にログインしてください」と案内して終了。パスワードはAI不可侵）
3. 対象ファイルが `/mnt/user-data/outputs/` 配下にあること（file_upload の制約。
   ワークスペースからは必ず outputs へコピーしてから添付する）
4. **消費の確認**: Claude Design の生成はモデル消費が大きい。画面に週次リミット警告
   （例: 「You've used 83% of your weekly limit」）が出ていたらユーザーに残量を伝えて
   続行可否を確認する。**何でも送らない — 人間が手を入れる価値のある完成品のみ**

## 手順（UIルート）

1. **ゲート**: 変更操作（プロジェクト作成）のため /タスク開始 のワークフロー下で実行
2. 成果物を outputs へコピー（PPTX 1ファイル推奨。PNG群なら6枚まで＋意図を1行）
3. claude.ai/design を開く → 「+」→ file input を find で特定し **file_upload**（refに直接。
   ネイティブファイルダイアログは開けない）
4. プロンプト入力（型）:
   「添付◯◯を**デザインを変えずに編集可能な状態で読み込んで**。
    配色 #……×#……とレイアウト維持」 → 再生成させない指示が重要
5. 送信 → 処理開始を確認（CP-1: プロジェクトURLが `claude.ai/design/p/<id>` に遷移した
   スクショ + URL控え）。処理完了まで待たなくてよい（生成は数分かかる）
6. **ハンドオフ**: タブをそのプロジェクトで開いたままにし、ユーザーに報告:
   「Claude Design に載せました → <URL>。キャンバスで直してください。
    終わったら『Design から回収して』で引き取ります」
7. 記録: knowledge/artifacts-index.md に URL を追記（種別: Design編集中）、
   session-log に1行

## 回収（人間の編集が終わったら）

トリガー: ユーザーの「Designから回収して」「編集終わった」

- **DesignSync 認可あり**: `list_files`→`get_file` で編集後ファイルを引き出し、
  ワークスペースの正本を更新（増分・1コンポーネントずつ）
- **UIルートのみ**: プロジェクトを開き、エクスポート/ダウンロード導線をUIで実行して
  outputs に回収 → 正本を更新。エクスポート導線が見つからない場合はユーザーに
  「Send to Claude Code Web」の実行を依頼する（こちらからは押さない — 共有系操作のため）
- 回収後: 差分を1行要約してユーザーに報告し、artifacts-index の状態を
  「Design編集中 → 回収済み」に更新。学び（人間がどこを直したか）を
  knowledge/feedback/lessons.md に記録し次回の生成品質に反映する — これが本部品の核心価値

## 禁止・注意

- 編集中のプロジェクトに AI が上書き介入しない（人間の編集領域。再度明示されるまで触らない）
- Share・公開リンク発行はユーザー承認なしに押さない
- リミット警告が出た状態での連投禁止（1セッション1〜2件まで）
- CP証跡なしで「送付成功」と記録しない（URL遷移スクショが証跡）
