# 部品: Canva 書き出し（旧 /キャンバ）

Canva の作業は **コネクタ（MCP）を第一経路、ブラウザ操作を補助経路**とする二段構え。エディタ内部の編集をブラウザで頑張らない（canvas 描画のため snapshot に構造が出ず、座標クリック頼みで壊れやすい）。

## 手順

### 1. 経路の確認

1. Canva 系ツール（`mcp__canva__*` 等）の有無を確認する（遅延ロードの可能性があるため検索・再試行してから判断）
2. **コネクタあり** → ステップ2へ。**なし** → ユーザーに「Cowork/Claude の設定から Canva コネクタを接続すると安定して操作できます（チャットからは認証できません）」と1行案内し、承認を得てブラウザ経路（ステップ3）で可能な範囲だけ実施する

### 2. コネクタ経路（推奨）

生えているツールの実名と入出力を確認し、できる操作（デザイン検索・作成・テンプレート流し込み・オートフィル・エクスポート）をツールで実行する。
初回はツール一覧と「できたこと/できなかったこと」を `knowledge/sites/canva/index.md` に記録する（次回の経路判断の正本）。

- 素材の流し込みは asset-collect 部品の `knowledge/assets/` + 出典台帳（クレジット要否）を使う
- 文言はドメインに応じた references/ スキル（copywriting 等）を適用し、公開向けは ad-compliance-jp を通す

### 3. ブラウザ経路（補助 — コネクタ未接続 or コネクタ非対応の操作のみ）

**向く操作**: テンプレート検索・デザイン一覧の確認・共有リンク取得・エクスポート（PNG/PDF ダウンロード）・素材アップロード。
**やらない操作**: エディタ内のレイアウト編集・ドラッグ操作（テキスト差し替え程度まで。それ以上はユーザーに委ねる）。

- 変更を伴う操作は /タスク開始 のゲート下で。初回は Canva の UI をフェーズ①でマッピングし `knowledge/sites/canva/` に記録
- エクスポートのダウンロードは**専用DLフォルダ**推奨（Downloads 本体はワークスペースに接続しない — 個人書類の露出防止。imagegen 部品（docs/parts/imagegen.md） の取り込み表と同じ原則）。専用フォルダが無ければ回収方法（手動移動）を明示する

### 4. 成果の記録

- 作成/書き出したデザインの共有URL・保存先を報告し、`knowledge/artifacts-index.md` に1行追記
- Canva の残クレジット・プラン制約に気づいたら `knowledge/sites/canva/index.md` に記録

## 注意

- **課金・プラン変更・Canva Pro へのアップグレードは AI は行わない**（料金ページは URL Guard / Money Watch の対象。発火したら復帰手順に従う）
- チーム共有デザインの削除・上書きは destructive として必ず事前承認を取る
