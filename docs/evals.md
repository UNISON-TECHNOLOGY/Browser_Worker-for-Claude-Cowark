# 評価ハーネス（golden タスク集）— 回帰テストの正本

スキル・サブエージェントの「腕が落ちていないか」を確認する固定タスク集。**委譲プロンプトの書き方**: サブエージェントに渡す SKILL.md 参照は絶対パスで明記し「Glob 探索は不要、その Read を直接実行」と指示する（2026-07-23 検証で deliverable-writer が実在する ad-compliance-jp/SKILL.md を「見つからない」と誤報告して停止する flake が発生。再ランでパス直指定により成功）。**LLM 実行が必要なため CI では回せない** — /検証 full の V27 で Cowork 実機（または Claude Code）から回す。PASS 基準は一意に書く（満たす/満たさないが割れない基準）。最終判定は基準に沿って LLM/人間が行う — 「機械的」とは判定者の裁量を残さないという意味で、grep 等の完全自動判定ではない。

## 運用

- 実行結果は `knowledge/verification/<date>-evals.md` に G1〜G7 の PASS/FAIL 表で記録
- FAIL したらスキル/エージェント本体を直し、直した内容を TESTING.md に記録（golden タスク自体を緩めて直すのは禁止）
- 新しい失敗事例（人間の差し戻し・実運用の見逃し）が出たら golden タスクに追加する（評価駆動: タスク集は育てる）
