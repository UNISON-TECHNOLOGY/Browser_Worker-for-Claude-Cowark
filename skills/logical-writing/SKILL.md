# ロジカルライティング スキル
**対象領域: 無形商材 × 人材業界**

データと論理で意思決定を支援する。採用戦略の提案、市場分析レポート、KPI報告、経営層への報告資料など。

## Use this skill when
- 採用戦略提案書をピラミッド構造やMECEで作成するとき
- KPIレポート（月次/週次）を作成するとき
- 候補者推薦レポートに論理的根拠を付けるとき
- 市場分析・競合比較・ROI試算等のデータドリブンな資料を求められたとき

## Do not use this skill when
- 感情に訴えるストーリー記事を求められたとき（-> storytelling）
- 求人票のライティングを求められたとき（-> recruit-writing）
- セールス目的のLP・スクリプトを求められたとき（-> sales-writing）
- 短いコピーを求められたとき（-> copywriting）

## 🤖 System Prompt（AIへの指示前提）
```text
あなたは事実とデータに基づき、説得力のある提案が行える優秀なコンサルタントです。
感情論ではなく論理的な構造（ピラミッド構造やMECE）を用いて、
クライアント企業や経営層が「なるほど、それなら投資（採用）すべきだ」と納得する論理展開を構築します。
```

## なぜ人材業界にロジカルライティングが必要か

> **無形商材は「信頼」で売る。信頼の基盤は「論理」。**

- 「なんとなく良い候補者です」→ 信頼されない
- 「3つの観点で評価し、貴社の課題にフィットする根拠は...」→ 信頼される
- 感情（ストーリー）× 論理（データ）の両輪が最強

## 基本原則

### 1. 主張は1つに絞る
- 1文書=1主張
- 「結局何を言いたいの？」に30秒で答えられること

### 2. 主張→根拠→具体例の三層構造
```
主張:  「採用チャネルをダイレクトリクルーティングに移行すべき」
 ├─ 根拠1: 「採用単価が40%下がる」
 │   └─ 具体例: 同業A社の実績データ
 ├─ 根拠2: 「候補者の質が上がる」
 │   └─ 具体例: 定着率比較データ
 └─ 根拠3: 「採用チームの工数が削減される」
     └─ 具体例: 工数比較表
```

### 3. MECE（漏れなく、ダブりなく）

## 人材業界のロジカル文書テンプレート

### 採用戦略提案書
```markdown
## 1. エグゼクティブサマリー
[結論: 何を提案し、どんな効果が見込めるか — 3行以内]

## 2. 現状分析
### 採用実績（直近12ヶ月）
| 指標 | 現状 | 業界平均 | 目標 |
|---|---|---|---|
| 採用充足率 | 65% | 78% | 90% |
| 採用単価 | 120万円 | 80万円 | 60万円 |
| 平均採用期間 | 45日 | 30日 | 21日 |
| 1年後定着率 | 72% | 80% | 90% |

### 課題の構造化
```
充足率が低い
├─ 応募数が少ない
│   ├─ 求人媒体の選定ミス
│   └─ 求人票の訴求力不足
├─ 選考辞退が多い
│   ├─ 選考スピードが遅い
│   └─ 面接体験の質
└─ 内定辞退が多い
    ├─ 年収交渉の失敗
    └─ 競合オファーに負ける
```

## 3. 提案内容
### 施策一覧と優先順位
| 施策 | 効果 | 工数 | 優先度 |
|---|---|---|---|
| 求人票リライト | 応募数1.5倍 | 低 | ★★★ |
| 選考フロー短縮 | 辞退率-30% | 中 | ★★★ |
| スカウト強化 | 質の向上 | 高 | ★★☆ |

## 4. 投資対効果（ROI）
### 現状コスト vs 提案後コスト
- 現状: 年間採用コスト [X]万円（[Y]名採用）
- 提案後: [X']万円（[Y']名採用）
- **削減効果: [Z]万円/年**

## 5. リスクと対策
| リスク | 影響度 | 発生確率 | 対策 |
|---|---|---|---|

## 6. スケジュール
| Phase | 期間 | 内容 |
|---|---|---|
| Phase1 | 1-2週目 | 求人票改善 |
| Phase2 | 3-4週目 | 選考フロー改善 |
| Phase3 | 2ヶ月目〜 | スカウト運用開始 |

## 7. 次のステップ
[具体的なアクション]
```

### 候補者推薦レポート（論理的根拠付き）
```markdown
## 推薦: [候補者名]（[職種]ポジション）

### 推薦の結論
貴社の[ポジション]に最適な候補者です。
3つの観点から根拠を説明します。

### 評価

#### 1. スキルマッチ（◎）
| 貴社の要件 | 候補者の実績 | マッチ度 |
|---|---|---|
| 法人営業3年以上 | 法人営業5年、リーダー経験 | ◎ |
| SaaS業界経験 | SaaS企業で2社経験 | ◎ |
| マネジメント | 5名チームのリーダー | ○ |

#### 2. カルチャーフィット（○）
- [具体的な行動特性や価値観の一致点]
- [面談時の所感 — 事実ベース]

#### 3. 転職動機と志向性（◎）
- 転職理由: [具体的に]
- 志向: [貴社のポジションとの接点]
- リスク要因: [正直に記載]

### 想定年収と条件
- 現年収: [X]万円
- 希望年収: [Y]万円
- 当社提案: [Z]万円（根拠: [市場相場データ]）

### 懸念点と対策
| 懸念 | 対策 |
|---|---|
| [懸念1] | [対策の提案] |
```

### KPIレポート（月次/週次）
```markdown
## [期間] KPIレポート

### サマリー
[結論を1-2行で: 何が順調で、何が課題か]

### KPI推移
| 指標 | 目標 | 実績 | 達成率 | 前月比 |
|---|---|---|---|---|
| 紹介数 | 30名 | 28名 | 93% | +12% |
| 面接設定率 | 60% | 55% | 92% | -3% |
| 内定率 | 25% | 30% | 120% | +8% |
| 入社確定 | 5名 | 4名 | 80% | ±0 |

### So What?（数字の解釈）
- 面接設定率が低下 → **原因**: [要因分析]
- 内定率が向上 → **要因**: [成功要因]

### 改善アクション
| アクション | 担当 | 期限 | 期待効果 |
|---|---|---|---|
```

## 論理の誤りチェックリスト（人材業界版）

- [ ] **「優秀な人材です」に根拠はあるか？**（何をもって優秀？）
- [ ] **サンプルサイズは十分か？**（1名の成功事例で全体を語っていないか）
- [ ] **相関と因果を混同していないか？**（離職率が低い≠この施策のおかげ）
- [ ] **暗黙の前提は正しいか？**（「人材紹介が最適」は本当か？）
- [ ] **競合との比較は公平か？**（自社に有利な比較軸だけ選んでいないか）
