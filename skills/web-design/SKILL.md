# Webデザイン/LP設計スキル

モックアップ生成・改善提案・デザインレビューで適用する実装原則。design-artisan / design-critic はこのスキルを基準として使う。

## 📚 リソース

- `resources/lp-cro.md` — CVR最適化の数値根拠（単一CTA・5秒テスト・フォーム項目数・不安解消）。改善提案の根拠付けに必読
- `resources/verify-checklist.md` — 出荷前チェックリスト（Critical/High/Medium）。design-critic の審査基準
- `resources/ux-laws.md` — UX心理学の根拠層（ヒック/フィッツ/フォン・レストルフ等）。提案に原則名を添えるために使う
- `resources/implementation.md` — 実装規範（レスポンシブ・画像・素材加工・a11y）。design-artisan のコーディング基準

## 1. レイアウト

- **1画面1メッセージ**: 各セクションは伝えることを1つに絞る。2つあるなら分割
- **視線の流れ**: Z型（ファーストビュー）→ 縦一直線（以降）。重要度と表示順を一致させる
- **余白は8pxグリッド**: 4/8/16/24/32/48/64/96px のみ使う。セクション間 ≥ セクション内 × 2（近接の原則）
- **コンテナ幅**: 本文 640〜760px、フル幅セクション内でもテキストは読み幅を守る。LP全体は 1040〜1200px
- **モバイルファースト**: 375px 幅で設計してから広げる。カード3カラムは必ず1カラムに畳む

## 2. タイポグラフィ

- **スケールを固定**: 1.25倍スケール推奨（12.8 / 16 / 20 / 25 / 31 / 39 / 49px）。中間値を発明しない
- **本文**: 16px以上、行間1.6〜1.8、1行 全角35〜42字
- **見出し**: 行間1.2〜1.4、太字(700)。サイズ差ではなく「サイズ+太さ+色」の3点で階層を作る
- **フォントは2種まで**: 見出し用+本文用。日本語は Noto Sans JP / ヒラギノ系が安全牌

## 3. カラー

- **60-30-10**: ベース60% / セカンダリ30% / アクセント10%。アクセントはCTA専用に温存する
- **CTAの色はページ内で唯一無二**: 他の要素にアクセント色を使わない（視覚的希少性がクリックを作る）
- **グレーは彩度を持たせる**: 純粋グレー(#999)より、ブランド色に寄せた無彩色（青みグレー等）
- **コントラスト**: 本文 4.5:1 以上、大きな見出し 3:1 以上（WCAG AA）

## 4. コンポーネント

- **ボタン**: 押せると分かる（影 or 塗り）、パディング 12-16px × 24-32px、hover/focus状態必須。ラベルは動詞（「登録する」not「登録」）
- **カード**: 影は1段階だけ（多段影の乱用禁止）、角丸はページ内で統一（4/8/12pxのどれか1つ）
- **フォーム**: 項目は最小限（1項目減るごとにCVR向上）、ラベルは上置き、エラーはインライン表示

## 5. LP構成（CRO）

標準骨格（必要に応じて削る。足すのは慎重に）:
1. **ヒーロー**: ベネフィット見出し（7-13字コピー）+ 実績/権威 + 主CTA。3秒で「誰向けの何か」が分かること
2. **共感（課題提起）**: ターゲットの悩みを3つ前後、自分ごと化
3. **解決策/ベネフィット**: 機能ではなく変化を語る（Before→After）
4. **証拠**: 実績数値・事例・お客様の声。具体的な数字+顔+名前が最強
5. **不安解消**: FAQ / 保証 / 「しないこと」の宣言
6. **行動**: 流れの提示（3-5ステップ）+ 最終CTA

原則:
- **CTAは同一文言・同一色で反復**（ヒーロー直下・中間・最下部）
- **数値は比較対象と併記**（「年収140万UP」>「年収UP」）
- **1スクロール内に必ず次を読む理由**（好奇心の断絶を作らない）

## 6. 禁止事項

- Lorem ipsum・ダミー画像のまま納品
- 装飾目的のアニメーション（意味のあるトランジションのみ、200-300ms）
- 3色以上のアクセント、3種以上のフォント、不統一な角丸
- 根拠（計測データ・診断）に紐付かないデザイン変更
