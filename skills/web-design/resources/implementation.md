# 実装ベストプラクティス（web.dev 系知見の再構成）

design-artisan がモックアップHTMLを書くときの実装規範。

## レスポンシブ

- ブレークポイントはコンテンツ基準で最小限（目安: 640 / 960px）。デバイス名基準にしない
- レイアウトは flexbox/grid + `minmax()` / `auto-fit` で流動化し、メディアクエリを減らす
- 流体タイポグラフィ: 見出しは `clamp(最小, vw式, 最大)`（例: `clamp(24px, 5vw, 40px)`）
- タッチターゲットは最小 44×44px、隣接ターゲット間 8px 以上

## 画像

- `width` / `height` 属性を必ず指定（CLS防止）
- ファーストビュー外の画像は `loading="lazy"`、ヒーロー画像は lazy にしない（LCP悪化）
- `max-width:100%; height:auto` を全画像に。背景画像より `<img>` を優先（alt・遅延・LCP計測のため）
- 提供画像が重い場合は圧縮・リサイズしてから埋め込む（下記「画像素材の扱い」参照）

## 画像素材の扱い（ユーザー提供画像）

1. 受け取った画像はまず `knowledge/assets/<site>/` に保存（元ファイルは上書きしない）
2. 加工は Bash の Python（Pillow）で行う: リサイズ（表示幅の2倍まで）、JPEG品質80前後 / PNG→WebP変換、サムネイル生成
3. モックアップへの埋め込みは相対パス参照を基本とし、アーティファクト発行時のみ data URI 化（外部URL参照は禁止）
4. 人物写真は改変（合成・補正で印象を変える加工）をしない。トリミングと圧縮のみ
5. 埋め込み時は alt 必須、装飾目的なら `alt=""`

## パフォーマンス

- Google Fonts は使用ウェイトのみ読み込み（`display=swap`）。それ以外の外部リソース禁止
- CSS はインライン1箇所に集約。未使用スタイルを残さない
- アニメーションは `transform` / `opacity` のみ（layout を揺らすプロパティ禁止）、200〜300ms

## セマンティクス / a11y

- ランドマーク: `header / main / footer / nav / section`（div 積みにしない）
- 見出しは h1 を1つ、h2→h3 の順序を飛ばさない
- フォーカス可視化（`:focus-visible` にアウトライン）、キーボードだけで全操作可能に
- 色だけで意味を伝えない（アイコン・テキストを併用）
