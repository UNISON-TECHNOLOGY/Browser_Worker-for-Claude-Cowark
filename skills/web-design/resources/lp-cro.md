# LP/CVR最適化リファレンス（数値根拠つき）

各種公開研究・事例から再構成した判定基準。design-artisan / design-critic が改善提案の根拠として使う。

## CTA

- **単一CTA原則**: CTAが1種類のLPは複数種のLPよりCVRが高い傾向（約13.5% vs 10.5%の調査例）。ページ内のリンク・ボタンは主CTA1種+補助（電話等）1種まで
- CTAラベルは成果を語る動詞句（「無料で診断を受ける」>「送信」）
- 反復配置: ヒーロー直下・証拠セクション後・最下部。文言と色は完全統一

## ファーストビュー

- **5秒テスト**: 5秒で「誰向けの・何の・どんな得」が答えられるか
- アウトカム型見出し（機能ではなく結果）は+35〜40%の改善報告例あり
- 最重要訴求をファーストビューへ移動して CVR +104% の国内事例。スクロールしないと主訴求が見えない構成は重大欠陥として扱う
- 広告・流入元とのメッセージ一致（訴求のズレは直帰の主因）

## フォーム

- **項目は3〜4個まで**を基準とする。1項目減らすごとに完了率は向上する
- 迷ったら「1ページ1設問」方式（GOV.UK 実証パターン）: 認知負荷が最小、エラー処理が単純、モバイルで崩れない
- ラベルは入力欄の上、プレースホルダをラベル代わりにしない、エラーは該当欄の直近にインライン表示
- 任意項目は削除する（「任意」と書くくらいなら聞かない）

## 不安解消（日本市場で特に効く）

- 返金・返品・キャンセル条件の明示（返品リンク設置で+46%の事例）
- 「しないこと」の宣言（例: 大量応募しない、しつこい電話をしない）は信頼形成に有効
- 実績数値は「n+単位+期間」まで具体化（「多数」は無意味）

## 改善の運用

- 変更は1回に1仮説（複数同時変更は学習にならない）
- 仮説は「ヒートマップ/計測 → 課題 → 打ち手 → 期待効果」の形式で記録し、knowledge/ に残す
