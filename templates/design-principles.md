# レポート設計原則

出典: デジタル庁「ダッシュボードデザインの実践ガイドブック」（2026-03-31版）の要点を
Delvework レポート（提示型ダッシュボードに相当）へ適用したもの。
deliverable-writer は HTML レポート執筆前にこのファイルを読むこと。

## 1. 情報の配置 = 視線の流れ

- **左上から右下へ、全体から部分へ**。冒頭に最重要サマリー（stat タイル）、下に行くほど詳細
- 最初に見るべき大きな指標を最上部に、判断や行動に役立つ細かい指標・表を後半に
- ページを分けず1枚で完結。長くなる場合はセクション番号で階層を明示

## 2. 数値は必ず比較対象と併記する

- 数量単体では意思決定できない。**前回比・目標値・業界目安のどれかを必ず添える**
- 例: LCP は実測値 + 閾値（2.5s）+ 前回計測との差分
- 増減の表現: 改善 = `.delta.up`（緑）、悪化 = `.delta.down`（赤）、目安超過 = stat の `.warn` / `.bad`

## 3. 情報量を絞る

- 目的（そのレポートで読者が下す判断）に照らして不要な計測値は載せない
- 「取ったデータを全部見せる」は禁止。詳細は JSON（データファイル）に残し、レポートは判断材料に絞る
- 要望・網羅性より一貫性。1レポート=1目的

## 4. グラフの選び方（CSSだけで表現する場合も原則は同じ）

| 伝えたいこと | 使う表現 |
|---|---|
| 時間変化・傾向 | 折れ線（または推移表 + delta） |
| 数量の比較 | 横棒（CSS の width%バー で可） |
| 構成比 | 積み上げバー。円は総量が明確な時だけ |
| 詳細の一覧 | 表（.tbl 必須）。行数が多ければ上位N + 「他X件」 |

- 軸・単位・データ時点（「YYYY-MM-DD時点」）を必ず明記
- 色数は系列数だけ。装飾目的の多色使いをしない

## 5. カラー（デジタル庁パレット準拠・コントラスト検証済み）

チャート系列色（テンプレートの CSS 変数に定義済み、この順で使う）:
`--chart-1` #3460FB (Blue600) → `--chart-2` #00A3BF (Cyan600) → `--chart-3` #259D63 (Green600)
→ `--chart-4` #FB5B01 (Orange600) → `--chart-5` #767676 (Gray)

セマンティック: 成功/改善 #197A4B、エラー/悪化 #CE0000、警告 #B58900。
白背景に対し 600番台は 3:1 以上を確保済み。文字として使う場合は 900 番台
（例 Blue900 #0017C1）を使う。

## 6. 鮮度と出典の明示

- ヘッダー meta に**計測日時**を必ず入れる。データが古い場合はその旨を明記
- 実測値と推定値を視覚的に区別する（`.badge.est` = 目視推定バッジ）
- フッターにデータソースのファイルパスと「次タスクへの導線」を書く

## 数値の根拠

本ファイルは統一骨格の正本。個別の数値判断（フォント・行長・コントラスト・CTA・グラフ選択）の実証根拠は `references/design-evidence-jp/SKILL.md` を参照。
