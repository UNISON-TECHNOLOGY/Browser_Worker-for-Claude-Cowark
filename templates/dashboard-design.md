# ダッシュボード用スタイルガイド — 「秘湯紀行」テーマ（正本）

`dashboard-template.html` の設計意図。**ダッシュボード（司令塔ページ）はこのテーマ**、
ガイド類は `guide-template.html`（和風総本家）、単発データレポートは `report-template.html`。

## コンセプト（2026-07-22 ユーザー指定）

> 仕事にゆとりを。安心した進捗で休日に愁いを残さない。

定常業務をAIに任せて自分の秘湯の旅（夕日・温泉・金の楼閣・日本庭園・遅咲きの桜）へ出たくなるページ。

## 大原則（2026-07-22 ユーザーフィードバックで確定。違反したら差し戻し）

1. **言葉で世界観を表現しない**。「表現がストーリーに合わせすぎて全く意味がわからない。
   言葉を使って表現するのは表現者として5流」— ラベル・見出し・単位・日付は**平易な機能名のみ**
   （アラート/サマリー/進行中タスク/SNSストック/競合ウォッチ/学習記録/生成物…）。
   帳場・報せ・道中・蓄えの升・見廻り・稽古帳・蔵・口上・漢数字章番号・「文月廿二日」等の言い換えは全面禁止
2. **世界観は視覚だけで表現する**: ヘッダー帯の絵、和紙の質感、金の罫、明朝体、夕日の配色、控えめな動き
3. **ファーストビューで価値に到達する**: 絵は高さ 230〜320px のヘッダー帯まで。
   開いた瞬間（スクロールなし）にアラートとサマリー数値が見えること。
   全画面ヒーロー・物語の幕・スローガンだけの1画面目は「全く価値がない」— 禁止

ダーク変種を追加しないこと。空の暈しは浮世絵の「ぼかし」表現として許容（本文側の面グラデは禁止のまま）。

## 型（レイアウト語彙）

- **ヘッダー帯 = 浮世絵ランドスケープ + 歩く旅人**（2026-07-22 ユーザー提供の参考画像に基づく。
  タイトル札の箱は「ダサい」と差し戻し済み — 箱を置かずタイトルは空に直書き。タブ帯のスクロールバーは非表示）:
  clamp(220px,30vh,300px)。クリーム（和紙色）の空 / 藍(#38445C)の富士+雪冠 / 金箔の月（箔片+暈し）/
  **青海波の雲**（SVG pattern、朱系と金系の2種、ゆっくり漂う）/ 尾根の道（点線）。
  すべて筆のにじみフィルタ（feTurbulence+feDisplacementMap）でべた塗りを殺す
- **旅人キャラクター（ブラウザワーカー）**: 丁髷・藍の道中着・金の帯・朱の襟・脚絆・草鞋・杖・
  背負い籠（茣蓙+短冊つき）のSVGキャラ。**タブを進めると尾根道を歩いて次の停留点へ移動**
  （transform transition 2s + 脚/腕/体の歩行アニメ、戻るときは反転。reduced-motion で停止）。
  道中の目印 = タブ対応: 道標(全体)→松林→洞窟→橋→灯籠→温泉+楼閣（最終タブは必ず温泉+楼閣）。
  通過済み停留点は金の点で塗る。
  **タブ数が雛形と異なる場合**は `stops` 配列と `.taboji` をタブ数と同数・等間隔に作り直す（唯一許可される構造変更）
- **攻略済みサイトの峰**: 右上の峰に金線の社を「攻略済みサイト数」だけ並べ、木の道標「攻略済み」を立てる
  （データと連動する遊び。実運用では社の数を knowledge/sites/ の攻略済み数に合わせる）
- **タブは「事業ドメイン単位」で切り、中に「場所 → タスク」を内蔵する**（2026-07-22 確定。
  変遷: タスクをフラットに並べる→「媒体の中にスカウトがあるはず。階層がおかしい」→
  場所単位→「その機能は内蔵だろ」→ ユーザー指定の4ドメインで確定）:
  「全体」+ 企画リサーチ（競合ウォッチ/自社サイト診断=構想抽出）/ SNS運用（レポート/投稿/ストック）/
  求人媒体（媒体ごとの掲載・スカウト）/ 広告分析（LP/動画分析）。
  実運用ではワークスペースの運用ドメインに合わせて増減する。
  各ドメインタブの中は「場所（サイト/媒体）ごとの節」: sec-head=場所名+タスク、状態カード、
  場所内の小分類は .naka-dai（金の短罫つき小見出し）で畳み込む。フェーズ①〜④は場所の攻略度。
  タブ札は sticky・明朝・活性は金の上罫。チップ: 朱=要対応件数、金=承認待ち件数。tablist + 矢印キー対応
- **システムの可視化（各タスクタブに必須）**: これがこのダッシュボードの本体
  - **状態カード**（.joho 4升）: フェーズ（①初回〜④最適化の札）/ 現在ステップ / 次回実行（スケジュール）/ 状態
  - **工程の飛び石図**（.kotei）: A指令→B外部探索→C内部探索→Dマップ→E変更前記録→F計画→G実行→H承認→I検証→Jレポート→K成果報告。
    済=苔色で塗り、現在=金で点灯、未=輪郭のみ。E と H の下に関所チップ（記録済=緑 / 承認待ち=朱）—
    「勝手に送信しない」ゲートを絵で示す
  - **承認待ちキュー**: 承認されるまで送信しない旨を袖注に明記し、承認の言い方（クリックコピー）を添える
  - **タスク固有KPI** と **このタスクの学習（OK/NG）**
- **「全体」タブ**: アラート → タスク一覧表（タスク/フェーズ/現在ステップ/次回実行/状態。タスク名クリックで該当タブへ）→
  直近7日の実行 → 蓄積と成長 → 生成物
- **タブ切り替えでヘッダーの場面が旅をする**: 全体=山 → スカウト送信=森 → 競合ウォッチ=洞窟 → 媒体管理=川 →
  SNS運用=庭園 → サイト診断=秘湯。物語は場面の遷移だけで語り、文字にしない
- **本体**: 和紙 feTurbulence 地。セクション=明朝見出し+金箔下線+右端に平易な補足。アラート（朱左罫）を「今日」タブ最上部
- **遊びの装飾は本体の中に**（2026-07-22「ダッシュボードの中こそ遊びの装飾を入れられる」への対応。ただしラベルは機能名のまま）:
  ①節見出しの**紋モチーフ**（金の線画・円枠つき: 鈴/扇/飛び石/焚火/提灯/升/千鳥/松/筆/双葉/湯呑/風呂敷）
  ②金箔下線の右に**箔の欠片** ③面には**鮫小紋**（アラート地・バーの中） ④バッジは**判子風**にわずかに傾ける
  ⑤statタイルに**円相の透かし** ⑥フッターに**道程の一筆書き**（山→森→洞窟→秘湯。装飾のみ・語りなし）
  ⑦表の行ホバーは金の淡染め、指示文ホバーは朱
- **数の見せ方**: statタイルは明朝の大きな数字（出現時カウントアップ）。表はUDゴシック+`tabular-nums`で右揃え。
  実行履歴はバー、ストックは充足率バー（不足は朱）
- **次の一手**: 各セクション末に金の鉤括弧の指示文。**クリックでコピー**（「コピーしました」トースト）
- **動きの節度**: セクションは静かなフェード出現のみ。花びら・湯気はヘッダー帯内に限定。
  `prefers-reduced-motion` で全停止＋即時表示

## 書体

- 見出し・指示文・stat数字: 明朝（Shippori/ヒラギノ明朝/游明朝、weight 500〜700 — 細らせない）
- 表・日付・数値: BIZ UDPGothic 系 + `tabular-nums`
- 外部フォント・CDN・画像読込は一切なし

## 禁じ手

guide-design.md の禁じ手を継承（絵文字・角丸大・青紫・番号バッジ・外部読込・「〜でござる」調）。
加えて: 世界観語彙によるUIラベルの言い換え（大原則1）、全画面ヒーロー/物語スクロール（大原則3）、
本体へのダーク変種、自動再生音。
